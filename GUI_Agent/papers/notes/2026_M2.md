## One-line Summary
M2 用 internal memory 做 trajectory summarization、用 external memory 从 insight bank 检索可执行经验，在无需训练的前提下把长程 web agent 的上下文压缩与决策质量同时提升，使 Qwen3-VL-32B 在 WebVoyager 上从 57.8% 提到 74.0%、token 降约 57%，并在 OnlineMind2Web 上最高提升 22.3 个点（Abstract / Table 2-3, p.1, p.7）。

## Problem Setting
- **Core problem**: "handling long-horizon tasks remains a critical bottleneck" and full-context prompting leads to "Exorbitant Computational Cost" plus "Performance Degradation" (Abstract / Introduction, p.1)
- **Assumptions**: 长程任务历史可被压缩成 state updates；离线 successful trajectories 可整理成 insight bank；检索到的高层经验可以跨站点迁移（Section 1, 3, p.1-8）。
- **Insufficiency of existing approaches**: 当前 full-context strategy 会让历史 HTML / screenshot / action 文本快速膨胀，并触发 lost-in-the-middle 问题（Introduction, p.1）

## Core Method
- **Method overview**: M2 明确把 memory 分成两层。第一层是 Internal Memory，作者用 Dynamic Trajectory Summarization 把 agent 最近交互不断折叠成简洁状态更新，从而避免 prompt 中累积一长串冗余 observation。第二层是 External Memory，即 Insight Retrieval Augmentation，从离线 insight bank 中检索与当前 query 相似的策略性经验，让 agent 在决策时不只看历史，也看“别人以前在类似网页上吃过什么亏、什么做法更稳”。

  这两个 memory 分别承担 hindsight 与 foresight。Internal memory 负责让 agent 不丢上下文，External memory 负责把离线经验迁移进来，减少在复杂网站层级中的盲目试探。作者强调这是一种 training-free augmentation，因此重点不是改变 backbone，而是改变 long-horizon prompt 的信息结构。
- **Key Design Choices**:

| Design Decision | Author's Choice | Author's Rationale | Ablation Verified? |
|----------------|----------------|--------------------|--------------------|
| Internal memory | Recursive trajectory summarization | 压缩长链 observation，减轻 context window 压力 | Yes |
| External memory | Insight bank retrieval | 将高层 UI interaction rule 迁移到新任务 | Yes |
| Similarity function | SentenceTransformer retrieval | 比 TF-IDF / BM25 更能对齐 query intent | Yes（Table 4, p.8） |
| Training setup | Training-free augmentation | 避免额外 SFT/RL 成本 | Indirectly verified |

- **Core difference from prior work**: 不同于需要 SFT/RL 学出 memory 的方法，M2 把 memory augmentation 设计成 inference-time plug-in；同时它不是只有摘要 memory，而是摘要 + insight retrieval 双通道联合（Section 1, 3, p.1-8）。

## Key Results

| Benchmark | This Method | Strongest Baseline | Δ | Notes |
|-----------|------------|-------------------|---|-------|
| WebVoyager (Qwen3-VL-32B) | 74.0 | Same model normal 57.8 | +16.2 pts | Token 215.2k -> 92.3k (Table 2, p.7) |
| WebVoyager (Claude-3.7-Sonnet) | 84.5 | Same model normal 72.0 | +12.5 pts | Combined memory best (Table 2, p.7) |
| WebVoyager (Claude-Sonnet-4) | 86.0 | Same model normal 80.5 | +5.5 pts | Near OpenAI CUA level (Table 2, p.7) |
| OnlineMind2Web medium (Qwen3-VL-32B) | 52.52 | Same model normal 30.22 | +22.3 pts | Strongest improvement slice (Table 3, p.7) |
| OnlineMind2Web avg. (Qwen3-VL-32B) | 51.55 | Same model normal 31.96 | +19.6 pts | Tokens down 58.7% (Table 3, p.7) |

- **Key ablation findings**: 在 WebVoyager 上，similarity function 使用 SentenceTransformer 明显优于 TF-IDF、Word2Vec、BM25；历史窗口以 5 步最佳，过长到 7 反而下降；去掉 action history 的损失比去掉 visual feedback 更大，说明 action trace 对 trajectory continuity 更关键（Table 4, p.8）。
- **Failure cases**: 作者没有做单独 failure gallery，但指出 moderate temporal cues 必要、过长 history 会引入 contextual interference；这说明 memory 不是越多越好，而是要做 selective compression（Section 3.2.2 / Table 4, p.8）。

## Limitations
- **Author-stated limitations**: 论文没有独立 limitations 章节，但作者在分析中明确指出 "excessive historical length introduces contextual interference that can degrade agent reasoning"；同时 "increasing the number to seven leads to a performance decline, likely due to the introduction of redundant information that distracts the agent." (Section 3.3 / Table 4 analysis, p.8)。
- **My observed limitations**: 
> M2 已经非常接近当前主线，但它的核心仍是 insight retrieval，不是 failure-aware procedural memory。External insight bank 依赖大规模成功轨迹，说明 training-free 不等于 data-free；同时它主要缓解 long-horizon context management，对 grounding / execution error 本身没有直接修复机制，因此更适合被视为 A-1 的 strongest partial solution，而不是终点。
- **Experimental design gaps**: 缺少真正 out-of-distribution 网站评测；也未量化 insight bank 构建质量、规模与性能之间的边际收益。

### Gap Signals (extracted from this paper)
- Gap signal 1: current full-context agents suffer from "Exorbitant Computational Cost" and "Performance Degradation" (Introduction, p.1) → 长程任务中的 context management 仍是核心瓶颈。

> 这篇最大的价值是把“可执行经验比原始观察更有用”证明得很清楚。按当前主线，它支持 A-1 的表示方向，但也同时暴露出缺口：insight 还不够结构化，验证与长期更新机制也不充分。

### Reusable Elements
- **Methodology**: recursive summarization、insight bank、query-conditioned retrieval、memory cost reporting。
> 最值得复用的是 external insight 的表达方式，把成功轨迹提炼成“防坑提示 / 操作规则”，而不是保留 raw trajectory。若继续沿当前主线推进，这些 insight 还需要补上触发条件、失败信号和 revision rule。
- **Experimental design**: 同时报 success、avg.step、avg.token，很适合评估 memory augmentation 是否真的“省而不弱”。
