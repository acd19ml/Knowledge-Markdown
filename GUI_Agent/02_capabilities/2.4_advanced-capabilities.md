## 3.2.1 个性化服务（Personalized Services）

### 为什么重要

不同用户在使用应用和网页时有不同的偏好、习惯和需求。个性化能力使 Agent 能够：
- **快速识别用户意图**
- **执行更符合个人风格的操作**
- 提升整体用户体验

### 当前实现方式

**核心机制：记忆模块（Memory Module）**

目前个性化主要通过声明性记忆（Declarative Memory）实现：

| 系统 | 个性化实现 |
|------|-----------|
| **Friday** | 在声明性记忆中记录用户画像：对话风格偏好、工具使用习惯、音乐/视频偏好 |

> "Personalization is usually realized through the memory modules of the agents."

### 挑战

- 个性化与**隐私保护**之间的张力（个人设备上的敏感信息）
- 现有研究极为有限，仅 Friday 有初步探索

### Research Gap

**这是最明显的 Gap 之一**：在 GUI Agent 领域，个性化几乎是空白。相比之下，Agent_Memory 综述中整个"用户中心记忆（User-Centric Memory）"维度专门讨论这一问题。两个知识体系的交叉点正在此处。

## 3.2.2 Agent 协同（Agent Synergy）

### 核心思路

将复杂任务拆解为多个子任务，每个子 Agent 专注特定功能，通过协同提升整体效率和准确性。

### 代表系统

**UFO（Windows 桌面端）**：
```
用户指令
    │
AppAgent  ← 选择合适的应用程序
    │
ActAgent  ← 在选定应用中执行具体操作
    │
完成任务
```

**MMAC-Copilot（桌面端，6角色）**：
```
Planner    → 分解任务为粗粒度子任务
Viewer     → 细化子任务 + 执行
Programmer → 编程执行子任务
Video Analyst → 视频分析支持
Librarian  → 信息检索，提供基础知识
Mentor     → 评估执行结果 + 反馈 Planner
```
通过"团队协作链"（Team Collaboration Chain）减少知识盲区造成的幻觉。

**Mobile-Agent-v2（移动端，3角色）**：
```
Planning Agent  → 压缩历史操作 + 屏幕摘要 → 任务进度（纯文本格式）
                                              ↓
Decision Agent  ← 当前进度 + 当前屏幕状态 + Reflection 反馈 → 生成并执行动作
                                              ↑
Reflection Agent → 监测动作前后屏幕变化 → 判断是否成功 → 纠错建议
```

### 多智能体对比

| 系统 | 平台 | Agent 数量 | 协同机制 | 特点 |
|------|------|-----------|---------|------|
| **UFO** | Windows | 2 | 分层（选App + 执行） | 简洁高效 |
| **MMAC-Copilot** | Desktop | 6 | 团队协作链 | 减少幻觉，功能全面 |
| **Mobile-Agent-v2** | Mobile | 3 | 规划+决策+反思闭环 | 专为移动优化 |

### Research Gap

- 多智能体 GUI Agent 研究仍处于早期，仅上述3个系统
- **Agent 间通信协议**尚未标准化
- 多智能体系统的**推理成本**成倍增加（与 Section 6.1 的成本问题叠加）
- 缺乏**跨 Agent 的共享记忆机制**（相关：[Self_Evolve/04_co-evolution.md](../../Self_Evolve/04_co-evolution.md)）
