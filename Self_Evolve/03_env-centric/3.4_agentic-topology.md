# Agentic Topology Evolution

> Paper Section IV-D (pages 9–10)

---

## Core Concept

Agentic Topology Evolution treats the **structure of multi-agent systems (MAS)** itself as a learnable, optimizable component. Instead of manually designing agent roles, communication patterns, and team configurations, this paradigm evolves the *system architecture* through search, RL, or meta-learning.

> "Traditional MAS are built upon manually crafted workflows and predefined agent roles, resulting in rigid architectures. This paradigm emphasizes evolving the system structure as a learnable and optimizable component."

Three categories:

```
Agentic Topology Evolution
├── 1. Offline Architecture Search     (optimize before deployment)
├── 2. Runtime Dynamic Adaptation      (adjust during inference)
└── 3. Structural State Evolution      (optimize shared collective memory)
```

---

## 1. Offline Architecture Search

Design the MAS topology as an **offline optimization problem** — search for the best structure before deploying the system.

### Communication Topology Optimization

| System | Key Innovation |
|---|---|
| **GPTSwarm** [Zhuge et al., 2024] | Applies gradient-based updates to agent prompts and connectivity to maximize information flow |
| **MACNET** [Qian et al., 2024] | Large-scale DAG-based multi-agent framework; identifies *collaboration scaling law*: irregular topologies > regular as agent count increases |
| **AutoFlow** [Li et al., 2024] | Leverages RL feedback to fine-tune interaction workflows from abstract descriptions |

### Programmatic Definition Search

Moving beyond fixed templates to searching over the space of possible agent *code* definitions:

| System | Key Innovation |
|---|---|
| **AFLOW** [Zhang et al., 2024] | Formulates workflow optimization as Monte Carlo Tree Search (MCTS) over code space |
| **ADAS** [Hu et al., 2024] | Automated Design of Agentic Systems — meta-agent iteratively writes and reflects on system code, discovering designs outperforming human baselines |
| **MAS-GPT** [Ye et al., 2025] | Frames MAS design as end-to-end generation task — synthesizes executable systems from high-level user intents |

### Key Finding: Collaboration Scaling Law

MACNET identifies that in optimized MAS:
- **Irregular topologies** (optimized) outperform **regular structures** (ring, star, fully-connected) as agent count increases
- Optimal communication is task-specific, not one-size-fits-all

## 2. Runtime Dynamic Adaptation

Unlike offline methods, this paradigm dynamically adjusts **agent configurations and connectivity during inference** — balancing performance and cost.

### Role Generation

| System | Key Innovation |
|---|---|
| **AutoAgents** [Chen et al., 2023] | Assembles dynamic expert teams based on task requirements |
| **EVOAGENT** [Yuan et al., 2025] | Applies evolutionary operators (mutation) to generate specialized skills and agent roles |

### Communication Topology Adaptation

| System | Key Innovation |
|---|---|
| **G-Designer** [Zhang et al., 2024] | Decodes communication structures via graph neural networks during inference |
| **MaAS** [Zhang et al., 2025] | Samples architectures from a probabilistic supernet |
| **Adaptive Graph Pruning** [Li et al., 2025] | Prunes low-utility communication edges during inference |

### Hierarchical Reasoning

| System | Key Innovation |
|---|---|
| **ReMA** [Wan et al., 2025] | Decouples high-level meta-thinking from execution — meta-agent plans, executor acts |
| **MASS** [Zhou et al., 2025] | Alternates between local prompt refinement and global topological search |

---

## 3. Structural State Evolution

Optimizes the **collective memory architectures** shared across agents — evolving *what* agents share and *how* they share it.

### Information Quality Management

| System | Key Innovation |
|---|---|
| **SEDM** [Xu et al., 2025] | Scalable self-evolving distributed memory — autonomously prunes low-value entries to prevent context overflow |
| **EvoConfig** [Guo et al., 2026] | Self-evolving multi-agent systems for efficient environment configuration |

### Structured Collective Memory

| System | Key Innovation |
|---|---|
| **G-Memory** [Zhang et al., 2025] | Organizes collective experience into **hierarchical graphs** — enables retrieval at different abstraction levels |
| **Collaborative Memory** [Rezazadeh et al., 2025] | Manages information boundaries through evolving bipartite graphs — balances sharing and privacy |

### Latent-Space Interaction

| System | Key Innovation |
|---|---|
| **LatentMAS** [Zou et al., 2025] | Shifts multi-agent interaction into **latent space via hierarchical KV caches** — synchronizes cognitive states directly without text decoding overhead |

## Key Design Dimensions

When designing agentic topology, key choices:

| Dimension | Options |
|---|---|
| Communication | Fully connected / DAG / Sparse / Dynamic |
| Memory sharing | Private / Shared workspace / Hybrid |
| Specialization | Generalist agents / Specialist roles |
| Coordination | Centralized / Decentralized / Hierarchical |
| Adaptation timing | Offline (fixed) / Online (dynamic) |

---

## Offline vs Runtime Trade-offs

| Dimension | Offline Search | Runtime Adaptation |
|---|---|---|
| Flexibility | Low (fixed after deployment) | High (adjusts per task) |
| Compute cost | High (search), then zero at inference | Added inference overhead |
| Optimality | Better for known task distributions | Better for diverse/novel tasks |
| Complexity | Search space is combinatorial | Requires efficient routing |
