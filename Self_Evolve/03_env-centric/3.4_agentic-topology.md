# 智能体拓扑演化

> 论文第 IV-D 节（第 9–10 页）

---

## 核心概念

智能体拓扑演化将**多智能体系统（MAS）的结构**本身视为一个可学习、可优化的组件。该范式不是手工设计智能体角色、通信模式和团队配置，而是通过搜索、RL 或元学习来演化*系统架构*。

> "传统 MAS 建立在手工制作的工作流和预定义的智能体角色之上，导致架构僵化。该范式强调将系统结构作为可学习和可优化的组件来演化。"

三个类别：

```
智能体拓扑演化
├── 1. 离线架构搜索     （部署前优化）
├── 2. 运行时动态适应   （推理时调整）
└── 3. 结构状态演化     （优化共享集体记忆）
```

---

## 1. 离线架构搜索

将 MAS 拓扑设计为**离线优化问题** —— 在部署系统前搜索最佳结构。

### 通信拓扑优化

| 系统 | 关键创新 |
|---|---|
| **GPTSwarm**（Zhuge et al., 2024） | 对智能体提示和连接应用基于梯度的更新以最大化信息流 |
| **MACNET**（Qian et al., 2024） | 基于 DAG 的大规模多智能体框架；识别*协作扩展规律*：随智能体数量增加，不规则拓扑 > 规则拓扑 |
| **AutoFlow**（Li et al., 2024） | 利用 RL 反馈从抽象描述fine-tuning交互工作流 |

### 程序化定义搜索

超越固定模板，在可能的智能体*代码*定义空间上进行搜索：

| 系统 | 关键创新 |
|---|---|
| **AFLOW**（Zhang et al., 2024） | 将工作流优化表述为代码空间上的蒙特卡洛树搜索（MCTS） |
| **ADAS**（Hu et al., 2024） | 智能体系统自动化设计 —— 元智能体迭代地编写和反思系统代码，发现超越人类基线的设计 |
| **MAS-GPT**（Ye et al., 2025） | 将 MAS 设计框架化为端到端生成任务 —— 从高层用户意图合成可执行系统 |

### 关键发现：协作扩展规律

MACNET 发现，在优化的 MAS 中：
- 随着智能体数量增加，**不规则拓扑**（经过优化的）优于**规则结构**（环形、星形、全连接）
- 最优通信是任务特定的，没有一刀切的方案

## 2. 运行时动态适应

与离线方法不同，该范式在**推理过程中动态调整智能体配置和连接** —— 平衡性能和成本。

### 角色生成

| 系统 | 关键创新 |
|---|---|
| **AutoAgents**（Chen et al., 2023） | 根据任务需求组装动态专家团队 |
| **EVOAGENT**（Yuan et al., 2025） | 应用进化算子（变异）生成专用技能和智能体角色 |

### 通信拓扑适应

| 系统 | 关键创新 |
|---|---|
| **G-Designer**（Zhang et al., 2024） | 在推理期间通过图神经网络解码通信结构 |
| **MaAS**（Zhang et al., 2025） | 从概率超网中采样架构 |
| **Adaptive Graph Pruning**（Li et al., 2025） | 推理期间剪枝低效用的通信边 |

### 层次化推理

| 系统 | 关键创新 |
|---|---|
| **ReMA**（Wan et al., 2025） | 将高级元思考与执行解耦 —— 元智能体规划，执行者行动 |
| **MASS**（Zhou et al., 2025） | 在局部提示精炼和全局拓扑搜索之间交替 |

---

## 3. 结构状态演化

优化跨智能体共享的**集体记忆架构** —— 演化智能体共享*什么*以及*如何*共享。

### 信息质量管理

| 系统 | 关键创新 |
|---|---|
| **SEDM**（Xu et al., 2025） | 可扩展的自演化分布式记忆 —— 自主剪枝低价值条目以防止上下文溢出 |
| **EvoConfig**（Guo et al., 2026） | 用于高效环境配置的自演化多智能体系统 |

### 结构化集体记忆

| 系统 | 关键创新 |
|---|---|
| **G-Memory**（Zhang et al., 2025） | 将集体经验组织为**层次化图** —— 支持在不同抽象层次进行检索 |
| **Collaborative Memory**（Rezazadeh et al., 2025） | 通过演化的二分图管理信息边界 —— 平衡共享和隐私 |

### 潜在空间交互

| 系统 | 关键创新 |
|---|---|
| **LatentMAS**（Zou et al., 2025） | 通过**层次化 KV 缓存将多智能体交互转移到潜在空间** —— 直接同步认知状态，无需文本解码开销 |

## 关键设计维度

设计智能体拓扑时的关键选择：

| 维度 | 选项 |
|---|---|
| 通信 | 全连接 / DAG / 稀疏 / 动态 |
| 记忆共享 | 私有 / 共享工作空间 / 混合 |
| 专业化 | 通才智能体 / 专家角色 |
| 协调 | 集中式 / 去中心化 / 层次化 |
| 适应时机 | 离线（固定）/ 在线（动态） |

---

## 离线 vs 运行时的权衡

| 维度 | 离线搜索 | 运行时适应 |
|---|---|---|
| 灵活性 | 低（部署后固定） | 高（按任务调整） |
| 计算成本 | 高（搜索），推理时为零 | 增加推理开销 |
| 最优性 | 对已知任务分布更好 | 对多样/新颖任务更好 |
| 复杂性 | 搜索空间是组合式的 | 需要高效路由 |

## 与记忆的关联

智能体拓扑与**多智能体记忆操作**密切相关（见 [Agent_Memory/03_operations/multi-agent-operations.md](../../Agent_Memory/03_operations/multi-agent-operations.md)）：
- 结构状态演化 = 演化多智能体记忆架构
- LatentMAS = 应用于 MAS 的内部/潜态记忆基底
- G-Memory = 集体智能体的图结构语义记忆
