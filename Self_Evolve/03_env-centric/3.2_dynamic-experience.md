# Dynamic Experience Evolution

> Paper Section IV-B | Table II in paper

---

## Core Concept

Dynamic Experience Evolution treats the environment as a **gymnasium** — a source of feedback signals that enable the agent to refine its *strategies*, not just its knowledge. The agent learns *how to do* through trajectory analysis and experience accumulation.

> "The environment functions as a gymnasium to refine reasoning strategies through trajectory analysis — supporting learning **how to do**."

Three levels by when learning occurs:

| Level | When | Key Systems |
|---|---|---|
| **Offline Experience** | After deployment, from stored trajectories | AWM, AgentKB, SkillWeaver, COPS |
| **Online Experience** | During deployment, real-time feedback | WebRL, agent training via RL |
| **Lifelong Experience** | Across all deployments, continuous accumulation | ArcMemo, FLEX, Evolver |

---

## A. Offline Experience Evolution

Agents learn from **past trajectories** collected during deployment, then update their strategies offline.

### Workflow Template Induction

**Agent Workflow Memory (AWM)** [Wang et al., 2024]:
- Extracts reusable workflow templates from successful trajectories
- Stores as parameterized procedural memory
- Key insight: recurring task patterns → generalizable execution templates

**AgentKB** [Tang et al., 2025]:
- Leverages cross-domain experience for agentic problem solving
- Builds shared knowledge base from diverse task executions

### Skill Library Management

**SkillWeaver** [Zheng et al., 2025]:
- Web agents self-improve by discovering and honing reusable skills
- Skills are executable code extracted from successful interactions

**COPS** [Yang et al., 2024]:
- LLM agents with provable cross-task experience sharing
- Formal guarantees on skill transfer quality

### Programmatic Skill Induction

**Inducing Programmatic Skills** [Wang et al., 2025]:
- Induces programmatic skills for agentic tasks
- Skills represented as executable programs for composability

### Key Pattern: From Trajectories to Skills

```
Successful trajectory → Extract reusable pattern → Store as skill
                                                        ↓
                                               Future similar task
                                                        ↓
                                               Retrieve relevant skill
                                                        ↓
                                               Execute with adaptation
```

## B. Online Experience Evolution

Agents learn **while executing tasks** — updating strategies in response to real-time environmental feedback.

### RL-Based Online Learning

**WebRL** [Qi et al., 2024]:
- Trains LLM web agents via self-evolving online curriculum RL
- Uses task success/failure as reward signal
- Curriculum: starts easy, progresses to harder tasks as agent improves

**RAGEN** [Wang et al., 2025]:
- Understanding self-evolution via multi-turn RL
- Studies dynamics of how policies evolve through interaction

**Agent-R** [Yuan et al., 2025]:
- Training LM agents to reflect via iterative self-training
- Combines online interaction with offline reflection

### Search-Augmented Online Learning

**Search-R1** [Jin et al., 2025]:
- Train LLMs to reason and leverage search via RL
- Online learning from search result quality

**MemSearcher** [Yuan et al.]:
- RL for joint searching and memory management

### Key Challenge: Reward Design

Online experience evolution requires **reward signals** that are:
- Automatically computable (no human judges)
- Aligned with task quality
- Shaped to provide dense feedback (not just final success/failure)

Solutions:
- **Verifiable tasks**: Code compilers, math solvers
- **Process rewards**: Step-level feedback via PRMs
- **Self-evaluation**: Agent evaluates its own intermediate steps

## C. Lifelong Experience Evolution

Agents accumulate and generalize experience **across all past deployments** — building long-term, ever-growing capability.

### Key Systems

| System | Key Innovation |
|---|---|
| **ArcMemo** [Ho et al., 2025] | Abstract reasoning composition with lifelong LLM memory |
| **FLEX** [Cai et al., 2025] | Continuous agent evolution via forward learning from experience |
| **Evolver** [Wu et al., 2025] | Self-evolving LLM agents through an experience-driven lifecycle |
| **MemEvolve** [Zhang et al.] | Meta-evolution of agent memory systems |
| **SWE-Exp** [Chen et al., 2025] | Experience-driven software issue resolution |
| **RememberMe, RefineMe** [Cao et al., 2025] | Dynamic procedural memory framework for experience-driven evolution |

### Lifelong Learning Challenges

```
Challenge 1: Catastrophic Forgetting
  → Old experience overwritten by new training
  → Solution: Memory consolidation, selective retention

Challenge 2: Transfer vs Interference
  → Skills from one domain may hurt performance in another
  → Solution: Task-specific routing, hierarchical abstraction

Challenge 3: Distribution Shift
  → Environment changes over time
  → Solution: Continual adaptation, memory with timestamps

Challenge 4: Scalability
  → Infinite experience accumulation is infeasible
  → Solution: Compression, forgetting, hierarchical storage
```

### Lifelong Agent Memory Architecture Pattern

```
New Experience
     ↓
[Encoding] → Working Memory (recent)
     ↓
[Consolidation] → Episodic Store (specific events)
     ↓
[Generalization] → Semantic Store (abstract skills)
     ↓
[Pruning] → Remove stale/redundant memories
```

## Table II: Taxonomy of Dynamic Experience Methods

| Category | When | Signal | Generalization |
|---|---|---|---|
| Offline | Post-hoc | Stored trajectories | Task patterns → workflow templates |
| Online | Real-time | Environment feedback | Current task optimization |
| Lifelong | Continuous | All past interactions | Cross-task abstraction |

## Connection to Agent Memory

Dynamic Experience Evolution maps directly to **Episodic and Procedural Memory** (see [Agent_Memory/02_taxonomy/2.2_cognitive-mechanisms.md](../../Agent_Memory/02_taxonomy/2.2_cognitive-mechanisms.md)):

| Memory Type | Role in Dynamic Experience |
|---|---|
| Episodic | Stores specific past trajectories and outcomes |
| Procedural | Extracted skills and workflow templates |
| Semantic | Generalized knowledge from multiple episodes |

The **Offline Evolution** paradigm corresponds to **Episodic → Procedural consolidation** at batch intervals.
The **Online Evolution** paradigm corresponds to real-time working memory → episodic updates.
