# Static Knowledge Evolution

> Paper Section IV-A

---

## Core Concept

Static Knowledge Evolution treats the environment as a **knowledge base** — a source of information that supplements the agent's internal knowledge. The environment does not change based on agent actions; the agent evolves by improving *how* it retrieves and uses this static knowledge.

> "The environment serves as a database to bridge informational gaps and supports learning **what is**."

Two major paradigms:
1. **Agentic RAG** — Selective, adaptive retrieval augmented generation
2. **Deep Research** — Multi-step web search and synthesis

---

## A. Agentic RAG

### Basic RAG → Agentic RAG

**Traditional RAG**: Query → Retrieve → Generate (one-pass)

**Agentic RAG**: The agent actively decides *when*, *what*, and *how* to retrieve — retrieval is a learnable, adaptive action.

### Key Systems

| System | Key Innovation | Retrieval Strategy |
|---|---|---|
| **FLARE** [Jiang et al., 2023] | Active retrieval: retrieves only when uncertain | Uncertainty-triggered |
| **Self-RAG** [Asai et al., 2024] | LLM generates special tokens to decide when/how to retrieve and self-critique | Self-reflection tokens |
| **Think-on-Graph** [Sun et al., 2023] | Reasoning over knowledge graphs with LLM | Graph traversal |
| **Search-R1** [Jin et al., 2025] | Train LLMs to reason and leverage search engines via RL | RL-optimized search |
| **Search-o1** [Li et al., 2025] | Agentic search-enhanced large reasoning models | Interleaved reasoning + search |
| **DSPy** [Khattab et al., 2023] | Compiles declarative LM calls into self-improving pipelines | Programmatic retrieval |
| **R1-Searcher** [Song et al., 2025] | Incentivizes search capability in LLMs via RL | RL-trained search |

### Self-RAG: Adaptive Retrieval with Self-Critique

```
Query → [Retrieve?] → Retrieved docs → [Is relevant?] → [Critique answer?] → Output
          ↑ token         ↑ token           ↑ token           ↑ token
       (special tokens generated by LLM itself)
```

Self-RAG trains the model to generate special tokens that control:
- Whether to retrieve at all
- Relevance of retrieved passages
- Groundedness of generated response

## B. Deep Research

Deep Research extends agentic retrieval to **multi-step, long-horizon web research** — synthesizing information across many documents and searches.

### Core Pattern
```
Initial Query
    ↓
Decompose into sub-queries
    ↓
Iterative search + retrieval
    ↓
Synthesize intermediate findings
    ↓
Generate comprehensive report
```

### Key Systems

| System | Key Innovation | Environment |
|---|---|---|
| **MindSearch** [Chen et al., 2024] | Mimics human mind with multi-agent deep search | Web search |
| **WebThinker** [Li et al., 2025] | Empowers large reasoning models with deep research | Web + reasoning |
| **DeepResearcher** [Zheng et al., 2025] | Scaling deep research via RL in real-world environments | Real web |
| **WebVoyager** [He et al., 2024] | End-to-end web agent with large multimodal model | Real websites |
| **MARS** [Chen et al., 2025] | Co-evolving dual-system deep research via MARL | Web + multi-agent |
| **Open Deep Search** | Democratizing search with open-source reasoning agents | Web |

### Deep Research Quality Dimensions
- **Breadth**: Coverage of relevant sources
- **Depth**: Multi-hop reasoning across documents
- **Reliability**: Citation accuracy and provenance
- **Synthesis**: Coherent integration of diverse information

---

## Agentic RAG vs Deep Research

| Dimension | Agentic RAG | Deep Research |
|---|---|---|
| Scope | Single question | Complex research task |
| Retrieval rounds | 1–3 adaptive | Many (10+) |
| Output | Grounded answer | Comprehensive report |
| Key challenge | When to retrieve | How to plan research |
| Training signal | Retrieval quality | Research quality metrics |

---

## Key Challenge: Hallucination vs Over-Retrieval

```
Under-retrieval → Hallucination (making up facts)
Over-retrieval  → Context overflow + degraded reasoning
```

**Self-RAG's solution**: Train retrieval as a discrete decision → only retrieve when predicted to help.

**Deep Research solution**: Hierarchical planning → retrieve only what's needed for each sub-task.

## Connection to Memory

Static Knowledge Evolution is closely related to **Semantic Memory** (see [Agent_Memory/02_taxonomy/2.2_cognitive-mechanisms.md](../../Agent_Memory/02_taxonomy/2.2_cognitive-mechanisms.md)):
- RAG externalizes semantic memory to vector stores
- Deep research uses active workspace akin to working memory
- Self-RAG adds meta-cognition about when retrieval is needed
