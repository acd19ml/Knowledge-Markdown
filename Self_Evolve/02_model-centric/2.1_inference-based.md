# Inference-Based Self-Evolution (Test-Time Scaling)

> Paper Section III-A (pages 5–7)

---

## Core Concept

Inference-Based Self-Evolution improves agent performance **without updating model weights** — purely through smarter computation at inference time. Also called **Test-Time Scaling** or **Test-Time Compute Optimization**.

> "The key insight: more inference compute = better answers, up to a scaling law that rivals training-time improvements."

Three mechanisms, expanding computation along different dimensions:

| Mechanism | Dimension | Strategy |
|---|---|---|
| **Parallel Sampling** | Width | Generate many candidates, select best |
| **Sequential Self-Correction** | Depth | Iterate on single trajectory with self-critique |
| **Structured Reasoning** | Search | Explicit tree/graph exploration of reasoning paths |

---

## 1. Parallel Sampling

**Idea**: Generate N responses in parallel, then select the best via aggregation or verification.

### Majority Voting / Self-Consistency
- **Self-Consistency** [Wang et al., 2022]: Generate multiple CoT paths → take majority vote on final answer
- Works because: different reasoning paths often reach the same correct answer; errors tend to be inconsistent
- Best for: deterministic domains (math, code)

### Best-of-N with Verifier
- Generate N candidates → score with verifier/reward model → select highest-scoring
- **Large Language Monkeys** [Brown et al.]: Scaling inference compute with repeated sampling
- Key finding: success rate scales predictably with N (power law)

### Ensemble Fusion
- **LLM-Blender** [Jiang et al., 2023]: Pairwise ranking + generative fusion of multiple LLM outputs
- Combines complementary strengths of different samples

### Key Insight: Probability Redistribution
```
Single sample: High probability (overestimated) vs Low probability (underestimated)
                        ↓
Parallel sampling: Correct redistribution of probability mass
                   → Correctly identifies rare-but-right answers
```

## 2. Sequential Self-Correction

**Idea**: Generate a solution, evaluate it (self or via feedback), revise iteratively.

### Self-Critique / Reflexion
- **Reflexion** [Shinn et al., 2023]: Agent generates "verbal reinforcement" — natural language critique of previous attempt → feeds into next iteration
- **Self-Refine** [Madaan et al., 2023]: Iterative refinement loop with self-generated feedback
- **Self-Debug** [Chen et al., 2023]: Models debug their own generated code

### Tool-Augmented Correction
- **CRITIC** [Gou et al., 2023]: Uses external tools (search, code execution) to validate and correct responses
- Grounded feedback prevents self-reinforcing hallucination loops

### Verifier-Guided Correction
- **SelfCheckGPT** [Manakul et al., 2023]: Detects hallucinations by checking consistency across multiple samples
- Small models + strong verifier can self-correct [Zhang et al., 2024]

### Limitation
> "Self-correction is only as good as the agent's self-evaluation capacity. Without external grounding, errors can compound rather than resolve."

---

## 3. Structured Reasoning (Search-Based)

**Idea**: Explicitly represent reasoning as a search problem over a structured space of thoughts.

### Chain-of-Thought (CoT) Foundation
- **CoT** [Wei et al., 2022]: Linear sequence of reasoning steps before final answer
- Basis: intermediate steps guide model toward correct reasoning

### Tree Search Methods

| Method | Structure | Search Strategy |
|---|---|---|
| **Tree of Thoughts (ToT)** [Yao et al., 2023] | Tree | BFS/DFS over thought nodes |
| **Graph of Thoughts (GoT)** [Besta et al., 2024] | Graph | Arbitrary merging/forking |
| **Language Agent Tree Search (LATS)** [Zhou et al., 2023] | Tree | MCTS with LLM rollouts |
| **AlphaZero-like** [Feng et al., 2023] | Tree | Value function guidance |

### Deep Reasoning / Extended Thinking
- **Think step by step → Evolving deeper LLM thinking** [Lee et al., 2025]
- **DeepSeek-R1** [Guo et al., 2025]: Long chain-of-thought with RL training → extended internal reasoning process
- **Meta-CoT** [Xiang et al., 2025]: Learning *how* to think, not just thinking longer

### Process Reward Models (PRMs)
- **Let's Verify Step by Step** [Lightman et al., 2023]: Process rewards at each reasoning step (not just final answer)
- Critical for: long-horizon tasks where final reward is sparse
- Enables step-level credit assignment for RL

### Key Scaling Finding
> "Scaling LLM test-time compute optimally can be more effective than scaling model parameters." — Snell et al., 2024

```
Performance gain:
  Larger model (train-time)  ≈  More compute at inference (test-time)
  (for same compute budget)
```

## Test-Time Reinforcement Learning (TTRL)

An emerging hybrid: applying RL *at inference time* without full weight updates.

- **TTRL** [Zuo et al., 2025]: Test-time reinforcement learning using majority voting as reward
- **TTCS** [Yang et al., 2026]: Test-time curriculum synthesis for self-evolving

---

## Summary: Inference-Based Trade-offs

| Method | Compute Cost | Weight Update | External Signal | Reliability |
|---|---|---|---|---|
| Parallel Sampling | High (N×) | No | Optional (verifier) | High with verifier |
| Self-Consistency | High (N×) | No | No | Moderate |
| Reflexion/Self-Refine | Medium (iterative) | No | Optional | Context-dependent |
| Tree Search (ToT, LATS) | High (exponential) | No | Optional | High but costly |
| TTRL | High | Minimal | Soft (voting) | Moderate |
