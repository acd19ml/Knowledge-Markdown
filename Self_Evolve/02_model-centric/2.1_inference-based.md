# 基于推理的自演化（测试时扩展）

> 论文第 III-A 节（第 5–7 页）

---

## 核心概念

基于推理的自演化在**不更新模型权重**的情况下提升智能体性能 —— 纯粹通过推理时更智能的计算来实现。也称为**测试时扩展**或**测试时计算优化**。

> "核心洞察：更多的推理计算 = 更好的答案，直至与训练时改进相当的扩展规律。"

三种机制，沿不同维度扩展计算：

| 机制 | 维度 | 策略 |
|---|---|---|
| **并行采样** | 宽度 | 生成多个候选，选择最佳 |
| **顺序自我修正** | 深度 | 通过自我批评在单条轨迹上迭代 |
| **结构化推理** | 搜索 | 对推理路径进行显式的树/图探索 |

---

## 1. 并行采样

**思路**：并行生成 N 个响应，然后通过聚合或验证选出最佳答案。

### 多数投票 / 自我一致性
- **Self-Consistency**（Wang et al., 2022）：生成多条 CoT 路径 → 对最终答案取多数投票
- 有效原因：不同推理路径经常得出相同的正确答案；错误趋向于不一致
- 最适合：确定性领域（数学、代码）

### 带验证器的 Best-of-N
- 生成 N 个候选 → 用验证器/奖励模型打分 → 选择得分最高的
- **Large Language Monkeys**（Brown et al.）：通过重复采样扩展推理计算
- 关键发现：成功率随 N 可预测地扩展（幂律）

### 集成融合
- **LLM-Blender**（Jiang et al., 2023）：多个 LLM 输出的成对排序 + 生成式融合
- 融合不同样本的互补优势

### 核心洞察：概率重分配
```
单次采样：高概率（被高估）vs 低概率（被低估）
                    ↓
并行采样：概率质量的正确重分配
         → 正确识别罕见但正确的答案
```

---

## 2. 顺序自我修正

**思路**：生成一个解，评估它（自评或通过反馈），然后迭代修改。

### 自我批评 / Reflexion
- **Reflexion**（Shinn et al., 2023）：智能体生成"言语强化" —— 对前一次尝试的自然语言批评 → 输入下一次迭代
- **Self-Refine**（Madaan et al., 2023）：带自生成反馈的迭代精炼循环
- **Self-Debug**（Chen et al., 2023）：模型调试自己生成的代码

### 工具增强的修正
- **CRITIC**（Gou et al., 2023）：使用外部工具（搜索、代码执行）验证和修正响应
- 有依据的反馈防止自我强化的幻觉循环

### 验证器引导的修正
- **SelfCheckGPT**（Manakul et al., 2023）：通过检查多个样本的一致性来检测幻觉
- 小模型 + 强验证器可以自我修正（Zhang et al., 2024）

### 局限性
> "自我修正的效果只取决于智能体的自我评估能力。没有外部依据，错误可能累积而非消解。"

---

## 3. 结构化推理（基于搜索）

**思路**：将推理显式表示为结构化思维空间上的搜索问题。

### 思维链（CoT）基础
- **CoT**（Wei et al., 2022）：最终答案前的线性推理步骤序列
- 基础：中间步骤引导模型走向正确推理

### 树搜索方法

| 方法 | 结构 | 搜索策略 |
|---|---|---|
| **Tree of Thoughts (ToT)**（Yao et al., 2023） | 树 | 在思维节点上进行 BFS/DFS |
| **Graph of Thoughts (GoT)**（Besta et al., 2024） | 图 | 任意合并/分叉 |
| **Language Agent Tree Search (LATS)**（Zhou et al., 2023） | 树 | 带 LLM 模拟的 MCTS |
| **AlphaZero-like**（Feng et al., 2023） | 树 | 价值函数引导 |

### 深度推理 / 扩展思考
- **Think step by step → Evolving deeper LLM thinking**（Lee et al., 2025）
- **DeepSeek-R1**（Guo et al., 2025）：带 RL 训练的长思维链 → 扩展的内部推理过程
- **Meta-CoT**（Xiang et al., 2025）：学习**如何**思考，而不仅仅是思考更久

### 过程奖励模型（PRM）
- **Let's Verify Step by Step**（Lightman et al., 2023）：在每个推理步骤给出过程奖励（而非仅最终答案）
- 对以下场景至关重要：最终奖励稀疏的长时域任务
- 支持 RL 的步骤级信用分配

### 关键扩展发现
> "最优地扩展 LLM 测试时计算，可能比扩展模型参数更有效。" —— Snell et al., 2024

```
性能增益：
  更大的模型（训练时）≈  推理时更多计算（测试时）
  （相同计算预算下）
```

## Test-Time RL (TTRL)（TTRL）

一种新兴的混合方案：在推理时应用 RL，无需完整的权重更新。

- **TTRL**（Zuo et al., 2025）：使用多数投票作为奖励的Test-Time RL (TTRL)
- **TTCS**（Yang et al., 2026）：用于自演化的测试时课程合成

---

## 总结：基于推理的权衡

| 方法 | 计算成本 | 权重更新 | 外部信号 | 可靠性 |
|---|---|---|---|---|
| 并行采样 | 高（N×） | 否 | 可选（验证器） | 有验证器时高 |
| 自我一致性 | 高（N×） | 否 | 否 | 中等 |
| Reflexion/Self-Refine | 中等（迭代） | 否 | 可选 | 依赖上下文 |
| 树搜索（ToT, LATS） | 高（指数级） | 否 | 可选 | 高但昂贵 |
| TTRL | 高 | 极少 | 软性（投票） | 中等 |

## 代表性系统

| 系统 | 策略 | 领域 | 关键创新 |
|---|---|---|---|
| Self-Consistency | 并行 + 多数投票 | 数学/推理 | 采样多样性 |
| Reflexion | 顺序 + 言语反馈 | 通用 | 言语 RL 循环 |
| CRITIC | 顺序 + 工具验证 | 事实/数学 | 有依据的修正 |
| Tree of Thoughts | 树搜索 | 规划 | 深思熟虑的探索 |
| LATS | MCTS + LLM 模拟 | 复杂任务 | 价值引导搜索 |
| DeepSeek-R1 | 扩展推理链 | 数学 | RL 训练的长 CoT |
