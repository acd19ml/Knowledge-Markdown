# Training-Based Self-Evolution

> Paper Section III-B (pages 7–9) | Table I in paper

---

## Core Concept

Training-Based Self-Evolution improves agent capabilities by **updating model weights** using data generated by the agent itself or synthesized from existing knowledge — without relying on additional human annotation.

Two sub-paradigms:
1. **Synthesis-Driven Offline Evolution** — generate data from existing models, train offline
2. **Exploration-Driven Online Evolution** — interact with environment during training, learn from live feedback

---

## A. Synthesis-Driven Offline Evolution

> "An efficient bootstrapper that consolidates internal priors — but fundamentally bounded by the base model's initial capacity."

The agent synthesizes its own training data through various strategies, then trains on this data offline.

### Table I: Five Categories of Offline Training Methods

#### 1. Instruction Synthesis
Generate diverse instruction-response pairs from the model itself.

| System | Key Idea | Notable Achievement |
|---|---|---|
| **Self-Instruct** [Wang et al., 2023] | LLM generates its own instruction dataset via prompting | Foundation of data-efficient alignment |
| **LIMA** [Zhou et al., 2023] | Less is more — 1000 curated examples rivals massive datasets | Quality > quantity for alignment |
| Instruction tuning survey [Zhang et al., 2026] | Comprehensive taxonomy | — |

**Key insight**: "SFT memorizes, RL generalizes" [Chu et al., 2025] — instruction tuning produces robust recall but limited generalization.

#### 2. Process Distillation
Extract reasoning processes from stronger models (or from the model itself via bootstrapping).

| System | Key Idea |
|---|---|
| **STaR** [Zelikman et al., 2022] | Bootstrap reasoning: generate rationales → filter correct ones → fine-tune |
| **Self-Play Fine-Tuning (SPIN)** [Chen et al., 2024] | Weak-to-strong: self-play converts weak models to strong through iterative SFT |
| **ReST-MCTS*** [Zhang et al., 2024] | Self-training via process reward guided tree search |

#### 3. Synthetic Augmentation
Expand training data beyond human-annotated boundaries.

| System | Key Idea |
|---|---|
| **Beyond Human Data** [Singh et al., 2023] | Scaling self-training beyond human-labeled data limits |
| **LADDER** [Simonds & Yoshiyama, 2025] | Recursive problem decomposition for self-improvement |
| Various scaling work | Data augmentation through diverse sampling strategies |

#### 4. Self-Play
Agents compete or collaborate with versions of themselves to generate training signal.

| System | Key Idea |
|---|---|
| **SPPO** [Wu et al., 2024] | Self-play preference optimization for alignment |
| **SPIRAL** [Liu et al., 2025] | Self-play on zero-sum games → incentivizes reasoning via MARL |
| **SPC** [Chen et al., 2025] | Evolving self-play critic via adversarial games |
| **ABSOLUTE ZERO** [Zhao et al., 2025] | Reinforced self-play reasoning with zero external data |

**Key insight**: Self-play creates diverse, auto-curriculated training signal without external data.

#### 5. Agentic Pipeline Evolution
Agents optimize their own prompts, workflows, and code pipelines.

| System | Key Idea |
|---|---|
| **DSPy** [Khattab et al., 2023] | Compile declarative LM programs into self-improving pipelines |
| **SIRIUS** [Zhao et al., 2025] | Multi-agent systems that bootstrap reasoning via self-improvement |
| **Agent Evolver** [Zhai et al., 2025] | Toward efficient self-evolving agent systems |

## B. Exploration-Driven Online Evolution

> "Transcends data ceilings by transforming the LLM into an active agent that discovers novel strategies through iterative trial-and-error."

The agent learns *while* interacting with the environment, generating its own reward signals through exploration.

### Key Mechanism: Environment as Training Ground

```
Agent → Takes action → Environment returns feedback
  ↑                              │
  └── Policy updated via RL ─────┘
      (GRPO, PPO, RLVR)
```

The environment provides **objective feedback** that prevents self-reinforcing biases and enables policies beyond initial training distribution.

### Reinforcement Learning Approaches

| Algorithm | Type | Key Property |
|---|---|---|
| **PPO** (Proximal Policy Optimization) | On-policy RL | Stable, widely used in RLHF |
| **GRPO** (Group Relative Policy Optimization) | Group-based RL | DeepSeek-R1's core algorithm |
| **DPO** (Direct Preference Optimization) | Offline RL | No reward model needed |
| **RLVR** (RL with Verifiable Rewards) | Verification-guided | Math/code domains |

### Key Systems

| System | Domain | Key Innovation |
|---|---|---|
| **WebRL** [Qi et al., 2024] | Web navigation | Self-evolving online curriculum RL for web agents |
| **WebAgent-R1** [Wei et al., 2025] | Web tasks | End-to-end multi-turn RL training |
| **RAGEN** [Wang et al., 2025] | Agent tasks | Understanding self-evolution via multi-turn RL |
| **Search-R1** [Jin et al., 2025] | Search | Train LLMs to reason and leverage search with RL |
| **R-Zero** [Huang et al., 2025] | Reasoning | Self-evolving reasoning LLM from zero data |

### Verifiable Reward Signal

A critical enabler: **automatically verifiable rewards** that don't require human judges.

Examples:
- **Math**: Symbolic solvers, Lean provers (AlphaProof)
- **Code**: Test suites, compilers, unit tests
- **Web tasks**: Task completion detection
- **Search**: Retrieved document relevance

> "External environments like code executors, mathematical engines, or the open web provide objective information — the environment acts as an essential pruning mechanism that prevents self-reinforcing biases."

## Key Tension: Offline vs Online

| Dimension | Synthesis-Driven Offline | Exploration-Driven Online |
|---|---|---|
| Data source | Synthesized / existing model | Live environment interaction |
| Efficiency | High (batch training) | Lower (sequential interaction) |
| Ceiling | Base model initial capacity | Environment complexity |
| Risk | Model collapse | Reward hacking |
| Safety | More controlled | Can discover exploits |
| Best for | Domain bootstrapping | Open-ended capability growth |

> "Offline serves as an efficient bootstrapper; Online transcends data ceilings by enabling discovery of novel strategies beyond training distribution."

## Representative Training Methods Summary

| Category | Method | Core Technique | Weight Update |
|---|---|---|---|
| Instruction Synthesis | Self-Instruct, LIMA | Self-generated instructions | SFT |
| Process Distillation | STaR, SPIN | Reasoning bootstrapping | SFT |
| Self-Play | SPPO, SPIRAL | Adversarial / cooperative | SFT/RL |
| Exploration-Driven | WebRL, RAGEN | Environment interaction | RL (PPO/GRPO) |
| Verification-Based | AlphaProof, TTRL | Formal verifiers | RL |
