# 动态经验演化

> 论文第 IV-B 节 | 论文表 II

---

## 核心概念

动态经验演化将环境视为**训练场** —— 一种使智能体能够精炼其*策略*（而不仅仅是知识）的反馈信号来源。智能体通过轨迹分析和经验积累学习**如何做**。

> "环境作为训练场，通过轨迹分析精炼推理策略 —— 支持学习**如何做**。"

三个层次，按学习发生的时机划分：

| 层次 | 何时 | 关键系统 |
|---|---|---|
| **离线经验** | 部署后，从存储的轨迹 | AWM、AgentKB、SkillWeaver、COPS |
| **在线经验** | 部署期间，实时反馈 | WebRL、通过 RL 的智能体训练 |
| **终身经验** | 跨所有部署，持续积累 | ArcMemo、FLEX、Evolver |

---

## A. 离线经验演化

智能体从部署期间收集的**过去轨迹**中学习，然后离线更新策略。

### 工作流模板归纳

**智能体工作流记忆（AWM）**（Wang et al., 2024）：
- 从成功轨迹中提取可复用的工作流模板
- 存储为参数化的程序记忆
- 核心洞察：反复出现的任务模式 → 可泛化的执行模板

**AgentKB**（Tang et al., 2025）：
- 利用跨领域经验进行智能体问题求解
- 从多样化任务执行中构建共享知识库

### 技能库管理

**SkillWeaver**（Zheng et al., 2025）：
- 网页智能体通过发现和磨练可复用技能来自我改进
- 技能是从成功交互中提取的可执行代码

**COPS**（Yang et al., 2024）：
- 具有可证明跨任务经验共享的 LLM 智能体
- 对技能迁移质量的形式化保证

### 程序化技能归纳

**Inducing Programmatic Skills**（Wang et al., 2025）：
- 为智能体任务归纳程序化技能
- 技能表示为可组合的可执行程序

### 关键模式：从轨迹到技能

```
成功轨迹 → 提取可复用模式 → 存储为技能
                                    ↓
                              未来相似任务
                                    ↓
                              检索相关技能
                                    ↓
                              执行并适配
```

## B. 在线经验演化

智能体在**执行任务的同时**学习 —— 根据实时环境反馈更新策略。

### 基于 RL 的在线学习

**WebRL**（Qi et al., 2024）：
- 通过自演化在线课程 RL 训练 LLM 网页智能体
- 以任务成功/失败作为奖励信号
- 课程：从简单任务开始，随智能体改进逐步升级到更难任务

**RAGEN**（Wang et al., 2025）：
- 通过多轮 RL 理解自演化
- 研究策略如何通过交互演化的动态过程

**Agent-R**（Yuan et al., 2025）：
- 通过迭代自我训练训练 LM 智能体进行反思
- 结合在线交互与离线反思

### 搜索增强的在线学习

**Search-R1**（Jin et al., 2025）：
- 通过 RL 训练 LLM 推理和利用搜索
- 从搜索结果质量中进行在线学习

**MemSearcher**（Yuan et al.）：
- 用于联合搜索和记忆管理的 RL

### 关键挑战：奖励设计

在线经验演化需要满足以下条件的**奖励信号**：
- 可自动计算（无需人类评判）
- 与任务质量对齐
- 能够提供密集反馈（不仅仅是最终的成功/失败）

解决方案：
- **可验证任务**：代码编译器、数学求解器
- **过程奖励**：通过 PRM 进行步骤级反馈
- **自我评估**：智能体评估自己的中间步骤

## C. 终身经验演化

智能体在**所有过去部署**中积累和泛化经验 —— 构建长期的、持续增长的能力。

### 关键系统

| 系统 | 关键创新 |
|---|---|
| **ArcMemo**（Ho et al., 2025） | 带终身 LLM 记忆的抽象推理组合 |
| **FLEX**（Cai et al., 2025） | 通过从经验的前向学习实现持续智能体演化 |
| **Evolver**（Wu et al., 2025） | 通过经验驱动的生命周期实现自演化 LLM 智能体 |
| **MemEvolve**（Zhang et al.） | 智能体记忆系统的元演化 |
| **SWE-Exp**（Chen et al., 2025） | 经验驱动的软件问题解决 |
| **RememberMe, RefineMe**（Cao et al., 2025） | 用于经验驱动演化的动态程序记忆框架 |

### 终身学习挑战

```
挑战 1：灾难性遗忘
  → 新训练覆写旧经验
  → 解决方案：记忆巩固、选择性保留

挑战 2：迁移 vs 干扰
  → 一个领域的技能可能损害另一领域的表现
  → 解决方案：任务特定路由、层次化抽象

挑战 3：分布偏移
  → 环境随时间变化
  → 解决方案：持续适配、带时间戳的记忆

挑战 4：可扩展性
  → 无限的经验积累是不可行的
  → 解决方案：压缩、遗忘、层次化存储
```

### 终身智能体记忆架构模式

```
新经验
     ↓
[编码] → 工作记忆（最近）
     ↓
[巩固] → 情节存储（具体事件）
     ↓
[泛化] → 语义存储（抽象技能）
     ↓
[剪枝] → 移除过时/冗余记忆
```

## 表 II：动态经验方法分类

| 类别 | 何时 | 信号 | 泛化 |
|---|---|---|---|
| 离线 | 事后 | 存储的轨迹 | 任务模式 → 工作流模板 |
| 在线 | 实时 | 环境反馈 | 当前任务优化 |
| 终身 | 持续 | 所有过去交互 | 跨任务抽象 |

## 与智能体记忆的关联

动态经验演化直接映射到**情节记忆和程序记忆**（见 [Agent_Memory_zh/02_taxonomy/2.2_cognitive-mechanisms.md](../../Agent_Memory_zh/02_taxonomy/2.2_cognitive-mechanisms.md)）：

| 记忆类型 | 在动态经验中的作用 |
|---|---|
| 情节记忆 | 存储具体的过去轨迹和结果 |
| 程序记忆 | 提取的技能和工作流模板 |
| 语义记忆 | 从多个情节中泛化的知识 |

**离线演化**范式对应于在批次间隔处的**情节 → 程序巩固**。
**在线演化**范式对应于实时的工作记忆 → 情节更新。
