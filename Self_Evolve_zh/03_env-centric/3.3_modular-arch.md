# 模块化架构演化

> 论文第 IV-C 节（第 9 页）

---

## 核心概念

模块化架构演化专注于优化**协调智能体-环境交互的结构模块** —— 不是交互的内容（静态知识）或获得的经验（动态经验），而是*机制*本身。

> "该范式专注于优化协调智能体-环境交互的结构模块 —— 提升交互效率和适应性。"

三个类别：

```
模块化架构演化
├── 1. 交互协议演化   （智能体如何管理上下文/历史）
├── 2. 记忆架构演化   （智能体如何组织存储的经验）
└── 3. 工具增强演化   （智能体如何创建/使用工具）
```

---

## 1. 交互协议演化

### 核心问题

长时域任务会生成溢出上下文窗口的大量交互历史。简单方法（拼接所有内容）因以下原因失败：
- Token 限制
- 上下文增长带来的边际收益递减
- 无关信息降低推理质量

### 认知抽象协议

这些方法将交互历史提炼为紧凑、有用的表征：

| 系统 | 关键创新 |
|---|---|
| **Think-in-Memory**（Liu et al., 2023） | 存储归纳性思考，避免推理步骤间的矛盾 |
| **Memory-of-Thought (MoT)**（Li & Qiu, 2023） | 过滤高置信度的推理路径以供复用 |
| **ReadAgent**（Lee et al., 2024） | 受模糊痕迹理论启发 —— 将文本压缩为"要点记忆"（关键思路，而非逐字） |

### 动态管理协议（OS 启发）

将 OS 级虚拟内存原理应用于智能体上下文：

| 系统 | 关键创新 |
|---|---|
| **MemGPT**（Packer et al., 2023） | 虚拟内存分页 —— 在"主内存"（上下文）和"外部存储"之间移动数据；LLM 充当 OS 内核 |
| **MemoryBank**（Zhong et al., 2024） | 纳入"艾宾浩斯遗忘曲线" —— 近期记忆更强，旧记忆衰减 |
| **AgentFold**（Ye et al., 2025） | 主动上下文折叠 —— 折叠无关轨迹以防止溢出 |
| **LightMem**（Fang et al., 2025） | 轻量高效的记忆增强生成 |
| **General Agentic Memory via Deep Research**（Yan et al., 2025） | 使用深度研究过程进行即时记忆重建 |

### 核心洞察：记忆作为主动重构

> "该范式不是被动积累原始交互历史，而是强调将轨迹提炼为简洁摘要和显著线索，调节在推理不同阶段应保留或暴露哪些信息。"

## 2. 记忆架构演化

### 从静态数据库到自适应系统

传统记忆 = 静态键值存储。演化的记忆 = 动态、自组织的网络，能够：
- 根据有用性决定存储什么
- 适应自身的组织结构
- 处理冲突或冗余的信息

### 关键系统

| 系统 | 关键创新 |
|---|---|
| **A-MEM**（Xu et al., 2025） | 将记忆视为动态网络的智能体记忆，灵感来自"卡片盒"笔记法 |
| **Mem0**（Chhikara et al., 2025） | 模仿人类认知智能地决定数据保留；生产级 |
| **MemEvolve**（Zhang et al.） | 元演化：智能体基于任务反馈自主适配路由策略和记忆逻辑 |
| **Memory as Action**（Zhang et al., 2025） | 自主上下文管理 —— 记忆管理作为可学习的决策过程 |
| **Mem-α**（Wang et al., 2025） | 通过强化学习学习记忆构建 |
| **CAM**（Li et al., 2025） | 建构主义方法：实时灵活同化认知模式 |

### 记忆作为强化学习决策

一种新兴方向：将记忆操作框架化为 RL 行动：

```
状态：当前工作记忆 + 查询
行动：{存储、检索、更新、压缩、遗忘}
奖励：任务性能提升
策略：学习的最优记忆管理策略
```

MemEvolve 和 Memory-as-Action 等系统采用这种方法，从启发式转向有原则的记忆控制。

## 3. 工具增强演化

### 进化路径：使用 → 创建 → 管理

```
级别 1：工具使用       ReAct、PAL（使用预定义 API）
级别 2：工具优化      WebGPT（学习何时使用哪个工具）
级别 3：工具创建      LATM、CREATOR（生成新工具）
级别 4：工具库管理    VOYAGER（持久可复用技能库）
```

### 级别 1-2：推理-执行协同

| 系统 | 关键创新 |
|---|---|
| **ReAct**（Yao et al., 2022） | 将推理（思考）与行动交叉 —— 基础模式 |
| **WebGPT**（Nakano et al., 2021） | 带人类反馈的浏览器辅助问答 |
| **PAL**（Gao et al., 2023） | 将复杂逻辑卸载到外部 Python 解释器 |

### 级别 3：自主工具创建

| 系统 | 关键创新 |
|---|---|
| **LATM**（Cai et al., 2023） | 从演示中为轻量级智能体生成工具 |
| **CREATOR**（Qian et al., 2023） | 将抽象设计与执行分离 —— 构建专用工具集 |
| **CRAFT**（Yuan et al., 2023） | 通过创建和检索专用工具集来定制 LLM |
| **TOOLMAKER**（Wölflein et al., 2025） | 将自主工具创建应用于带自动调试的科学工作流 |

### 级别 4：终身技能库

**VOYAGER**（Wang et al., 2023）：
- 将**成功动作存储为可执行代码**在技能库中
- 通过在 Minecraft 中的终身学习无限增长
- 比基线快 **15.3 倍**的进展

**Alita**（Qiu et al., 2025）：
- 通过极简预定义架构实现可扩展演化
- 通过标准化的上下文交互从紧凑核心动态生成工具
- 支持开放式能力扩展

**流程**：
```
任务 → 执行 → 成功？
  ↓ 是              ↓ 否
将技能抽象      调试/修改
进入库          策略
    ↓
未来相似任务检索技能
```

## 总结：三种子范式对比

| 子范式 | 演化对象 | 目标 | 关键挑战 |
|---|---|---|---|
| 交互协议 | 历史如何被管理 | 高效上下文使用 | 信息丢失 vs 溢出 |
| 记忆架构 | 经验如何组织 | 最优保留/检索 | 决定保留什么 |
| 工具增强 | 智能体可用的工具 | 扩展行动空间 | 工具质量/可靠性 |
