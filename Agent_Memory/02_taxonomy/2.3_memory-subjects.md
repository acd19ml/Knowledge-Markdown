# 记忆主体 —— 记忆服务于谁

> 论文第 3.3 节（第 17–20 页）

记忆主体定义了记忆系统被设计来捕获和支持**谁的信息**。该维度与记忆基底和认知机制均正交 —— 同一个外部向量存储可以服务于以用户为中心或以智能体为中心的目的。

---

## 两类主体

| 主体 | 记忆内容 | 主要目标 |
|---|---|---|
| **以用户为中心** | 用户偏好、身份、交互历史、对话上下文 | 跨会话为个体用户个性化智能体行为 |
| **以智能体为中心** | 任务轨迹、结果、学到的技能、领域知识 | 随时间提升智能体任务性能和能力 |

---

## 3.3.1 以用户为中心的记忆

> 旨在捕获和支持**用户信息**的记忆 —— 包括用户的偏好、历史和上下文。

### 核心应用场景

#### 长期个性化
- 跨会话和时间维护一致的用户档案
- 捕获稳定的偏好（沟通风格、领域专业知识、饮食限制等）
- 示例：A-Mem 构建随每次交互演化的动态用户记忆

#### 隐私保护记忆
- 通过访问控制和选择性共享保护存储的用户数据
- 挑战：平衡长期记忆的实用性与隐私要求
- 在医疗、法律和金融领域至关重要

#### 对话中的记忆管理
- 在长对话会话中保持用户相关上下文的一致性
- 防止早期上下文遗忘（用户姓名、偏好、既定目标）
- 示例：RoleLLM 维持角色一致性记忆；MemoCRS 跟踪用户偏好用于推荐

### 以用户为中心记忆的关键挑战

1. **偏好漂移**：用户随时间改变想法 —— 记忆必须适应，而不是冻结
2. **信用分配**：哪些过去的交互与当前请求相关？
3. **记忆过时**：何时应该覆盖旧偏好 vs 保留旧偏好？
4. **隐私与安全**：持久用户记忆 → 攻击面（记忆投毒、未授权访问）
5. **信号冲突**：用户说一套做一套 —— 存储哪个？

### 代表性工作

| 系统 | 关注点 |
|---|---|
| RoleLLM (Wang et al., 2024e) | 通过静态提示编码记忆保持角色一致性 |
| MAUMB (Hou et al., 2024) | 多用户、多智能体记忆管理 |
| MemoCRS (Xi et al., 2024) | 基于用户偏好记忆的对话推荐 |
| RET-LLM (Modarressi et al., 2023) | 用于长期对话的显式读写记忆 |
| Zep (Rasmussen et al., 2025) | 面向 AI 助手的生产级持久记忆 |
| A-Mem (Xu et al., 2025e) | 动态演化的用户记忆 |
| Echo (Liu et al., 2025i) | 面向个性化响应的以用户为中心记忆 |
| EpiCache (Kim et al., 2025b) | 用户交互的高效情节缓存 |

## 3.3.2 以智能体为中心的记忆

> 旨在捕获和支持**智能体自身信息**的记忆 —— 包括其任务历史、学到的技能和积累的专业知识。

### 核心应用场景

#### 长时域任务
- 存储关键中间状态，支持长期多步执行
- 避免重复已完成的工作；从中断处继续
- 示例：JARVIS-1 在 Minecraft 中维护已完成子任务的多模态记忆

#### 特定领域的长尾解决方案
- 现实世界问题呈现长尾现象 —— 罕见但重要的模式
- 智能体存储定制化的一次性解决方案，并在类似情况出现时复用
- 示例：
  - 软件调试：特定配置问题、依赖冲突
  - 科学研究：特定子领域的专门实验设置
  - 信息搜索：范围狭窄、依赖上下文的问题

#### 跨任务知识迁移
- 跨任务和情节积累持久知识
- 将与任务无关的知识抽象 → 泛化到新领域
- 关键区别：不是重放具体轨迹，而是提炼为**可迁移的抽象**
- 示例：Agent KB (Tang et al., 2025b) —— 汇聚高层策略的跨领域经验框架

#### 策略与技能学习
- 保留基于环境的程序记忆，编码**如何高效执行**多步行动
- 网页智能体：复用成功的多步浏览策略
- GUI 智能体：存储界面特定的行动序列（菜单导航、控件操作、错误恢复）
- 具身智能体：可执行技能 + 遵守物理动态的控制策略
- 示例：Voyager (Wang et al., 2025c) 技能库；JARVIS-1；AWM

### 以智能体为中心记忆的关键挑战

1. **存储什么**：并非所有轨迹都值得保留 —— 一次经历何时具有泛化价值？
2. **技能抽象**：如何从具体任务执行中提取可迁移知识
3. **遗忘有价值的信息**：随着记忆增长，如何在不丢失重要边缘情况解决方案的前提下剪枝
4. **跨领域迁移**：如何识别存储的策略何时适用于新领域

### 代表性工作

| 系统 | 关注点 |
|---|---|
| JARVIS-1 (Wang et al., 2024q) | Minecraft 具身智能体的多模态记忆 |
| Buffer of Thoughts / BoT (Yang et al., 2024b) | 存储程序性思维模板供复用 |
| AWM (Wang et al., 2025u) | 网页自动化的工作流记忆 |
| HippoRAG 2 (Gutiérrez et al., 2025) | 人类启发的情节索引用于智能体记忆 |
| Cognee (Markovic et al., 2025) | 智能体的knowledge graph记忆 |
| Branch-and-Browse (He et al., 2025b) | 网页探索的树结构行动记忆 |
| GridMM (Wang et al., 2023d) | 网格世界导航的空间记忆 |

## 以用户为中心 vs 以智能体为中心：关键差异

| 维度 | 以用户为中心 | 以智能体为中心 |
|---|---|---|
| **主要受益者** | 人类用户 | 智能体系统 |
| **存储内容** | 用户偏好、身份、对话历史 | 任务轨迹、技能、领域知识 |
| **时间范围** | 跨用户会话（周/月/年） | 跨任务实例和情节 |
| **成功指标** | 用户满意度、个性化质量 | 任务完成率、效率 |
| **隐私关注** | 高（个人身份信息、敏感偏好） | 较低（任务级数据） |
| **更新触发** | 用户交互和反馈 | 任务结果和智能体反思 |

---

## 同一系统中的两类主体

许多现实世界的系统**同时维护两者**：

```
基础智能体
    ├── 以用户为中心的记忆
    │   ├── 用户档案（姓名、偏好、历史）
    │   ├── 对话上下文（过去对话的关键点）
    │   └── 敏感用户数据（带访问控制）
    │
    └── 以智能体为中心的记忆
        ├── 任务轨迹（尝试了什么、什么有效）
        ├── 技能库（可复用的程序）
        └── 领域知识（积累的专业知识）
```

示例：LD-Agent (Li et al., 2025a) —— 同时维护长期用户交互记忆和智能体侧任务知识，用于对话系统。
