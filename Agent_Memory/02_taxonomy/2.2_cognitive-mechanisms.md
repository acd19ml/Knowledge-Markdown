# 认知记忆机制 —— 记忆服务于何种功能

> 论文第 3.2 节（第 12–19 页）

五种**原子认知记忆系统**，源自认知心理学，映射至基于 LLM 的智能体实现。

> "这五种系统构成了认知记忆最小且在架构上完备的分解。"

其他认知记忆构造（自传体记忆、前瞻记忆）可被理解为建立在这五种之上的组合。

---

## 汇总表（论文表 1）

| 认知类型 | 时间尺度 | 功能角色 | 智能体实例示例 |
|---|---|---|---|
| **感觉记忆** | 毫秒级 | 在进一步处理之前对原始感知输入的短暂保留 | 保留最近 2–5 秒的视频帧/传感器嵌入 |
| **工作记忆** | 秒至分钟 | 复杂任务所需信息的临时保持与主动操作 | 进行中的推理链："目标：精炼综述；早期章节确立了框架；下一次修订应保持框架一致性" |
| **情节记忆** | 小时至年 | 具体过往经历的情境化记录 | 过往交互日志："上次你更喜欢两页摘要；上一个计划因缺少 API 密钥而失败" |
| **语义记忆** | 持久 | 可跨任务使用的抽象事实和概念知识 | 知识库：项目信息、偏好、定义 —— 按查询检索，验证可靠性 |
| **程序记忆** | 持久 | 技能、例程和行动策略（隐式/可执行） | 可复用工作流："搜索 → 阅读 → 提取 → 引用"或"用消毒剂调试"，作为例程调用 |

## 3.2.1 感觉记忆

> "感觉记忆指对传入感知信号的临时保留，允许注意力和选择机制在更高层处理发生之前运作。"

### 在当前智能体中的体现

- **很少被显式建模** —— 文本输入将感知处理折叠为符号表征
- **在多模态/具身智能体中隐式实现**，表现为**短暂感知缓冲区**：
  - 视觉、音频或交互嵌入的缓存
  - 过滤/摘要/路由至工作记忆

### 研究方向

| 方向 | 功能 | 代表性工作 |
|---|---|---|
| 感知缓冲 | 在最近视频帧上维护短暂的嵌入队列 | HMT (He et al., 2025c)、LightMem (Fang et al., 2025a)、M2PA (Zhou et al., 2025b) |
| 流式 KV 记忆 | 在在线推理过程中保留最近的 token/视觉表征 | ReKV (Di et al., 2025)、V-Rex (Kim et al., 2025a)、SAM2 (Ravi et al., 2024) |
| 时序门控 | 为下游推理稳定嘈杂/高带宽的观测 | 各类具身智能体系统 |

### 为何将愈发重要

随着智能体在**多模态、具身、机器人**场景中部署：
- 必须实时处理连续高带宽流（视频、音频、本体感觉）
- 感觉记忆 → 缓冲 + 门控 → 减少冗余计算，稳定部分可观测性
- 实现原则性巩固：原始感知输入 → 工作记忆/情节记忆

## 3.2.2 工作记忆

> "工作记忆指一种短期记忆机制，支持复杂任务（如推理、理解和学习）所需信息的**临时存储和主动操作**。"

### 在智能体中的核心目标

在**严格的在线容量限制**下维持和操作任务相关状态，使多步推理能够不中断地进行。

由于 LLM **本质上是无状态的**，工作记忆在交互步骤间提供了显式的状态传递。

### 实例化方式

主要形式：**活跃context window**（提示上下文 + 中间推理链 + 工具输出 + KV 缓存）

**两条研究线**：

1. **预写压缩**（状态进入上下文之前）：
   - 压缩、重构或抽象交互历史，以延缓上下文饱和
   - 示例：用轻量引用替换大型中间输出 (Labate et al., 2025)；对已完成推理片段进行定期摘要/折叠 (Kang et al., 2025c; Wu et al., 2025d; Sun et al., 2025b)

2. **在线状态管理**（状态积累后）：
   - 在执行过程中，在固定在线预算下持续维护、更新或逐出状态
   - **智能体层面**：显式的记忆更新决策 (Zhang et al., 2025n; Yuan et al., 2025a)
   - **系统层面**：运行时 KV 缓存管理与调度 (Kim et al., 2025b; Liu et al., 2025j; Kwon et al., 2023)

### 核心洞察

> "现有工作表明，持续连贯的长时域推理不依赖于更长的context window，而依赖于在执行过程中**选择性地保留和操作任务相关状态**。"

工作记忆 = 智能体的**在线工作台**，而不仅仅是更大的缓冲区。

### 代表性工作

RAP (Hao et al., 2023)、FOT (Tworkowski et al., 2023)、ATFS (Christakopoulou et al., 2024)、MemReasoner、ChunkKV、LM2、ACON、FoldGRPO、MemoryAsAction、Kg-agent

## 3.2.3 情节记忆

> "情节记忆存储**情境化的经历记录**（发生了什么/在哪里/何时以及结果如何），供后续检索和复用。"

### 核心功能

1. **情节录制**：写入什么、如何组织事件/轨迹、多尺度情节形成
2. **检索与反思**：自适应触发、情节选择、塑造长期可访问性的保留策略

### 智能体层面的模式

```
经历发生
    ↓
结构化情节被记录（事件、结果、时间戳、上下文）
    ↓
存储在情节记忆中（向量存储/文本记录/图节点）
    ↓
当类似情境出现时被检索
    ↓
为当前决策提供参考
```

### 关键挑战：记录什么、如何存储

- 并非所有交互都值得记录
- 情节粒度：细粒度事件 vs 粗粒度情节摘要
- 检索触发器：何时查询过去的情节

### 代表性工作

Synapse (Zheng et al., 2024)、AgentCF (Zhang et al., 2024a)、WarAgent (Hua et al., 2023)、COMEDY、MemoryOS (Kang et al., 2025a)、Nemori、LearntoMemorize、PISA、MemoTime、DeepAgent、CAPEAM

**奠基性工作**：Generative Agents (Park et al., 2023) —— 带反思的记忆流，将高层洞察综合为关系和计划

---

## 3.2.4 语义记忆

> "语义记忆存储**稳定的抽象知识** —— 事实、概念和模式 —— 可跨任务使用，超越具体情节。"

### 核心功能

1. **知识归纳与组织**：记忆图谱、模式、紧凑神经表征
2. **知识访问与可靠性控制**：选择性激活、验证、在分布偏移下的持续修订

### 智能体层面的模式

```
遇到事实/概念/偏好
    ↓
被抽象并结构化为语义表征
    ↓
存储在语义存储中（knowledge graph/关系表/参数权重）
    ↓
按查询检索
    ↓
检查可靠性（幻觉检测、矛盾消解）
    ↓
跨不同任务用于推理
```

### 与情节记忆的关键区别

| | 情节记忆 | 语义记忆 |
|---|---|---|
| 内容 | 带情境的具体事件 | 抽象事实和概念 |
| 情境依赖性 | 植根于特定时间/地点 | 与上下文无关 |
| 示例 | "上周二 API 失败的会议" | "用户偏好简洁摘要；Python > Java" |

### 代表性工作

ChatDB (Hu et al., 2023)、FinMem (Yu et al., 2025e)、QuantAgent、LLMECM、Meminsight、CAM、PLTI、MoM (Zhao et al., 2025a)、Mem-PAL

## 3.2.5 程序记忆

> "程序记忆存储**可复用的技能、例程和工作流** —— 通过执行表现而非显式回忆来表达的可执行行动策略。"

### 核心功能

1. **技能归纳与封装**：从经验、工具或交互轨迹中学习可复用的程序
2. **技能执行、组合与适配**：在长时域的变化情境下调用和精炼程序

### 智能体层面的模式

```
成功的行动序列/工作流被观察到
    ↓
被抽象为可复用的技能/程序
    ↓
存储在程序记忆中（JSON 序列、代码、抽象工作流）
    ↓
当类似任务出现时被检索
    ↓
执行/适配到新情境
```

### 典型示例：Voyager (Wang et al., 2025c)

- Minecraft 智能体
- 将成功的行动存储为**可执行代码**在技能库中
- 随时间积累复合能力
- 每项新技能建立在之前技能的基础上

### 代表性工作

MetaGPT (Hong et al., 2023)、G-Memory (Zhang et al., 2025c)、MIRIX (Wang & Chen, 2025)、Memp (Fang et al., 2025b)、ReasoningBank (Ouyang et al., 2025)、MemGen (Zhang et al., 2025d)、BREW、ViReSkill

## 跨记忆类型交互

在实践中，这些记忆类型是**深度交织**的：

```
感知 →  感觉记忆
              ↓（过滤/选择）
          工作记忆（活跃上下文）
            ↙           ↘
       情节记忆         语义记忆
            ↘           ↙
            程序记忆
           （提炼出的技能）
```

**示例流程**（来自论文图 2）：
1. 用户请求规划东京之行 → 工作记忆保持当前任务状态
2. 智能体检索情节记忆（过去的日本旅行笔记）+ 语义记忆（用户食物偏好）
3. 生成计划 → 记录新情节
4. 随着时间推移，旅行规划技能成为程序记忆

## 研究空白

- **感觉记忆**：探索最不充分；对具身智能体的重要性日益增长
- **记忆巩固**：情节 → 语义 → 程序的转变在智能体中如何发生
- **程序记忆学习**：何时从情节经验中提炼技能（阈值如何确定？）
- **记忆干扰**：信息冲突时，多种记忆类型如何相互影响？
