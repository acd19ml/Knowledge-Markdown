# Cognitive Memory Mechanisms — What Function Memory Serves

> Paper Section 3.2 (pages 12–19)

Five **atomic cognitive memory systems** drawn from cognitive psychology, mapped to LLM-based agent implementations.

> "These five systems constitute a minimal and architecturally complete decomposition of cognitive memory."

Other cognitive constructs (autobiographical memory, prospective memory) can be understood as compositions built upon these five.

---

## Summary Table (Table 1 in paper)

| Cognitive Type | Time Scale | Functional Role | Agent Instantiation Example |
|---|---|---|---|
| **Sensory** | Milliseconds | Brief retention of raw perceptual inputs before further processing | Keep last 2–5 seconds of video frames / sensor embeddings |
| **Working** | Seconds–minutes | Temporary holding + active manipulation of current task state | In-progress reasoning chain: "goal: refine survey; earlier sections set framing; next revision should preserve framing consistency" |
| **Episodic** | Hours–years | Contextual record of specific past experiences | Past interaction log: "last time you preferred a 2-page summary; the previous plan failed due to missing API keys" |
| **Semantic** | Persistent | Abstract facts and conceptual knowledge usable across tasks | Knowledge base: project info, preferences, definitions — retrieved by query, checked for reliability |
| **Procedural** | Persistent | Skills, routines, and action policies (implicit/executable) | Reusable workflow: "search → read → extract → cite" or "debug with sanitizer", invoked as a routine |

## 3.2.1 Sensory Memory

> "Sensory memory refers to the temporary retention of incoming perceptual signals, allowing attention and selection mechanisms to operate before higher-level processing occurs."

### In Current Agents

- **Rarely explicitly modeled** — textual inputs collapse perceptual processing into symbolic representations
- **Implicitly realized** in multimodal/embodied agents as **short-lived perceptual buffers**:
  - Caches of visual, audio, or interaction embeddings
  - Filter/summarize/route to working memory

### Research Directions

| Direction | What It Does | Representative Works |
|---|---|---|
| Perceptual buffering | Maintain short-lived embedding queues over recent video frames | HMT (He et al., 2025c), LightMem (Fang et al., 2025a), M2PA (Zhou et al., 2025b) |
| Streaming KV memory | Retain recent tokens/visual representations during online inference | ReKV (Di et al., 2025), V-Rex (Kim et al., 2025a), SAM2 (Ravi et al., 2024) |
| Temporal gating | Stabilize noisy/high-bandwidth observations for downstream reasoning | Various embodied agent systems |

### Why It Will Matter More

As agents deploy in **multimodal, embodied, robotic** settings:
- Must process continuous high-bandwidth streams (video, audio, proprioception) in real-time
- Sensory memory → buffering + gating → reduces redundant computation, stabilizes partial observability
- Enables principled consolidation: raw sensory input → working/episodic memory

## 3.2.2 Working Memory

> "Working memory refers to a short-term memory mechanism that supports the **temporary storage and active manipulation** of information necessary for complex tasks such as reasoning, comprehension, and learning."

### Core Goal in Agents

Maintain and manipulate task-relevant state under **strict online capacity constraints** so multi-step reasoning can proceed without interruption.

Since LLMs are **inherently stateless**, working memory provides explicit state carrying across interaction steps.

### How It's Instantiated

Primary form: **active context window** (prompt context + intermediate reasoning traces + tool outputs + KV cache)

**Two research lines**:

1. **Pre-write compression** (before state enters context):
   - Compress, restructure, or abstract interaction history to delay context saturation
   - Examples: lightweight references replacing large intermediate outputs (Labate et al., 2025); periodic summarization/folding of completed reasoning segments (Kang et al., 2025c; Wu et al., 2025d; Sun et al., 2025b)

2. **Online state management** (after state has accumulated):
   - Continuously maintain, update, or evict state under fixed online budget during execution
   - **Agent-level**: explicit memory update decisions (Zhang et al., 2025n; Yuan et al., 2025a)
   - **System-level**: runtime KV cache management and scheduling (Kim et al., 2025b; Liu et al., 2025j; Kwon et al., 2023)

### Key Insight

> "Rather than treating longer context windows as the sole solution, existing work shows that sustaining coherent long-horizon reasoning depends on **selectively retaining and manipulating task-relevant state** during execution."

Working memory = the agent's **online workspace**, not just a bigger buffer.

### Representative Works

RAP (Hao et al., 2023), FOT (Tworkowski et al., 2023), ATFS (Christakopoulou et al., 2024), MemReasoner, ChunkKV, LM2, ACON, FoldGRPO, MemoryAsAction, Kg-agent

## 3.2.3 Episodic Memory

> "Episodic memory stores **contextualized experience records** (what/where/when and outcomes) for later retrieval and reuse."

### Core Functions

1. **Episode recording**: What to write, how to organize events/trajectories, multi-scale episode formation
2. **Retrieval and reflection**: Adaptive triggering, episode selection, retention policies shaping long-term accessibility

### Agent-Level Pattern

```
Experience occurs
    ↓
Structured episode recorded (event, outcome, timestamp, context)
    ↓
Stored in episodic memory (vector store / text-record / graph node)
    ↓
Retrieved when similar situation arises
    ↓
Informs current decision making
```

### Key Challenge: What to Record, How to Store

- Not all interactions deserve recording
- Episode granularity: fine-grained events vs. coarser episode summaries
- Retrieval triggers: when to look up past episodes

### Representative Works

Synapse (Zheng et al., 2024), AgentCF (Zhang et al., 2024a), WarAgent (Hua et al., 2023), COMEDY, MemoryOS (Kang et al., 2025a), Nemori, LearntoMemorize, PISA, MemoTime, DeepAgent, CAPEAM

**Foundational**: Generative Agents (Park et al., 2023) — memory stream with reflection to synthesize high-level insights

---

## 3.2.4 Semantic Memory

> "Semantic memory stores **stable abstract knowledge** — facts, concepts, and schemas — usable across tasks beyond specific episodes."

### Core Functions

1. **Knowledge induction and organization**: Memory graphs, schemas, compact neural representations
2. **Knowledge access and reliability control**: Selective activation, validation, continual revision under distribution shift

### Agent-Level Pattern

```
Facts / concepts / preferences encountered
    ↓
Abstracted and structured into semantic representations
    ↓
Stored in semantic store (knowledge graph / relational table / parametric weights)
    ↓
Retrieved by query
    ↓
Checked for reliability (hallucination detection, contradiction resolution)
    ↓
Used in reasoning across different tasks
```

### Key Distinction from Episodic Memory

| | Episodic | Semantic |
|---|---|---|
| Content | Specific events in context | Abstract facts and concepts |
| Context-dependence | Situated in time/place | Context-independent |
| Example | "Last Tuesday's meeting where API failed" | "User prefers concise summaries; Python > Java" |

### Representative Works

ChatDB (Hu et al., 2023), FinMem (Yu et al., 2025e), QuantAgent, LLMECM, Meminsight, CAM, PLTI, MoM (Zhao et al., 2025a), Mem-PAL

---

## 3.2.5 Procedural Memory

> "Procedural memory stores **reusable skills, routines, and workflows** — executable action policies expressed implicitly through performance rather than explicit recall."

### Core Functions

1. **Skill induction and packaging**: Learning reusable procedures from experience, tools, or interaction traces
2. **Skill execution, composition, and adaptation**: Invoking and refining procedures under changing contexts over long horizons

### Agent-Level Pattern

```
Successful action sequence / workflow observed
    ↓
Abstracted into reusable skill/procedure
    ↓
Stored in procedural memory (JSON sequences, code, abstract workflow)
    ↓
Retrieved when similar task arises
    ↓
Executed / adapted to new context
```

### Key Example: Voyager (Wang et al., 2025c)

- Minecraft agent
- Stores successful actions as **executable code** in a skill library
- Enables compounding capabilities over time
- Each new skill builds on previous ones

### Representative Works

MetaGPT (Hong et al., 2023), G-Memory (Zhang et al., 2025c), MIRIX (Wang & Chen, 2025), Memp (Fang et al., 2025b), ReasoningBank (Ouyang et al., 2025), MemGen (Zhang et al., 2025d), BREW, ViReSkill

## Cross-Memory Interactions

In practice, these memory types are **deeply intertwined**:

```
Perception →  Sensory Memory
                    ↓ (filtering/selection)
              Working Memory (active context)
                ↙           ↘
         Episodic         Semantic
         Memory           Memory
              ↘           ↙
              Procedural Memory
               (distilled skills)
```

**Example flow** (from Figure 2 in paper):
1. User asks to plan Tokyo trip → Working memory holds current task state
2. Agent retrieves episodic memory (past Japan travel notes) + semantic memory (user food preferences)
3. Generates plan → records new episode
4. Over time, travel planning skill becomes procedural memory

## Research Gaps

- **Sensory memory**: most under-explored; growing importance for embodied agents
- **Memory consolidation**: how episodic → semantic → procedural transitions happen in agents
- **Procedural memory learning**: when to extract skills from episodic experience (what threshold?)
- **Memory interference**: how do multiple memory types interact when information conflicts?
