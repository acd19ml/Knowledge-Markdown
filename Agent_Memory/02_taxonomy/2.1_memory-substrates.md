# 记忆基底 —— 记忆存储于何处

> 论文第 3.1 节（第 8–12 页）

记忆基底是在智能体与人类交互过程中，用于保留历史知识的 **存储机制**。分为外部和内部两大类。

---

## 外部记忆（§3.1.1）

> "外部记忆指将知识、信息和过往经验存储于智能体模型参数或状态**之外**的任何记忆基底。智能体可以通过检索和更新操作显式地读写外部记忆。"

### 核心特性

- 独立于神经网络权重运行（梯度更新不影响它）
- 清晰分离：**计算**（在 LLM 中）与 **知识**（在外部 DB 中）
- 可扩展且灵活：无需重新训练即可添加、替换或插入
- 以原始形式保留历史信息 → 减少幻觉，避免知识截断问题

### 不足之处

- 检索会引入**延迟**（在多轮迭代中尤为明显）
- 检索结果可能**不可靠**（噪声或无关结果带来干扰）
- 随着记忆增长，存储和索引成本**持续上升**
- 需要显式的记忆管理：摘要提炼、选择性保留、遗忘、去重、剪枝

### 四种实现类型

#### 1. 向量索引

**原理**：将记忆项嵌入到共享向量空间 → 近似最近邻搜索 → 将 top-k 条目追加到提示中。

**核心洞察**：将记忆视为**几何问题** —— 每个条目是连续空间中的一个点；查询找到语义最近的邻居。

**索引技术**：HNSW（层次可导航小世界）、IVF（倒排文件索引）、PQ（乘积量化）

**主导范式**：RAG（Retrieval-Augmented Generation (RAG)）框架

**代表性工作**：Lewis et al. (2020)，RAG 框架；大多数当前的智能体记忆系统

#### 2. 文本记录

**原理**：将记忆作为显式**文本文档** —— 持续更新的"核心摘要"+ 情节日志，定期汇总，选择性地复制到提示中。

**结构**：
- **语义列表**：离散事实，支持插入/更新/删除
- **情节列表**：带时间戳的事件按时间顺序记录
- **核心摘要**：持久的高层次摘要

**优点**：透明、快速集成、人类可读
**缺点**：需要细致的摘要提炼和剪枝

**代表性工作**：Generative Agents (Park et al., 2023)、SCM、MemGPT

#### 3. 结构化存储

**原理**：以**显式模式**存储记忆 —— 通过符号查询或遍历进行检索。

三种子类型：

| 类型 | 存储方式 | 检索方式 | 适用场景 |
|---|---|---|---|
| **关系表** | SQL 行 | SQL 连接、索引 | 事实、偏好、事件 |
| **图结构记忆** | 节点（实体）+ 边（关系） | 语义嵌入 + 关键词搜索 + 遍历 | 知识关系、事件关联 |
| **树结构记忆** | 带摘要 + 嵌入的层次节点 | 多层抽象 | 长对话任务、层次化知识 |

**knowledge graph细节**：节点 = 实体，边 = 关系。遇到新信息时：检测边的变化、消解冲突、维持有效图结构（Jiang et al., 2025c）。

**树结构细节**：每个节点存储聚合摘要 + 嵌入；越深的分支 = 越具体的细节。动态机制支持多层抽象 + 高效检索（Rezazadeh et al., 2025b, MemTree）。

**代表性工作**：ChatDB (Hu et al., 2023)、HippoRAG 2、Cognee、MemTree、PISA

#### 4. 层次化存储

**原理**：多个专用**记忆模块**，每个模块有自己的模式和策略，由元记忆管理器协调。

**模块示例**：
- 核心记忆：持久的人格 + 用户事实
- 情节记忆：时间序列、敏感事件
- 语义记忆：抽象概念
- 程序记忆：分步指令（JSON 序列）
- 资源记忆：文档、媒体
- 知识库：敏感/私密信息（严格访问控制）

**各模块检索方法**：嵌入相似度、BM25 匹配、字符串匹配

**元记忆管理器**：将查询导向正确的存储，汇总检索结果

**优点**：模块化、关注点分离、可扩展性强、专用模式
**缺点**：架构复杂、潜在延迟、数据一致性挑战

**代表性工作**：MIRIX (Wang & Chen, 2025)、MemGPT、MAUMB、多组件系统

## 内部记忆（§3.1.2）

> "内部记忆指直接存储在模型架构内部的信息，包括嵌入其参数中的持久性知识（参数化记忆）和推理过程中使用的工作状态。"

### 三种实现类型

#### 1. 权重（参数化记忆）

**原理**：通过训练或编辑，将知识/经验直接嵌入神经网络参数。智能体无需外部检索即可回忆信息。

**三种更新策略**：

| 策略 | 方法 | 核心思路 |
|---|---|---|
| **持续学习** | 增量权重更新 | 正则化 / 软掩码避免灾难性遗忘 |
| **模型编辑** | 有针对性的参数编辑 | 定位决定性的中间层权重 → 秩一更新（ROME、MEMIT） |
| **知识蒸馏** | 将上下文压缩为权重 | 训练学生模型复制带提示的教师行为；"上下文蒸馏" |

**优点**：高速访问、与推理紧密集成、无检索开销
**缺点**：更新成本高；有灾难性遗忘风险；编辑时存在干扰

**代表性工作**：MemoryLLM (Wang et al., 2024m)、SELF-PARAM、Titans、LMLM、MAC、WISE

#### 2. 潜在状态记忆

**原理**：在步骤/片段间传递和复用**中间隐藏状态**。即使原始 token 不再在窗口中，较早的激活值也会直接影响后续计算。

**关键特性**：在运行时激活中产生 → 使用 → 丢弃（除非被导出）。支持**会话内一致性**，但不跨会话持久化。

**研究方向**：
- 跨层重建隐藏表征以强化早期上下文
- 在片段间缓存隐藏状态向量的紧凑子集
- 生成记忆触发器 + 记忆编织器 token（合成潜态 token 反馈回输入）
- 对隐藏状态转换的结构化调制以维持语义对齐
- 集成到注意力机制中的压缩记忆（Munkhdalai et al., 2024）
- 推理时快权重更新（Zhu et al., 2025b）

**资源成本**：必须同时持有参数和中间状态；随模型深度、批次大小、精度线性扩展。

**代表性工作**：Titans (Behrouz et al., 2024)、HMT、MemGen、RET-LLM

#### 3. KV 缓存

**原理**：Transformer 推理时的记忆机制。缓存前序 token 的每层注意力键和值；后续 token 复用，而非重新计算。

**无 KV 缓存**：每一步为所有前序 token 重新计算 K/V 矩阵 → 长序列下 O(n²) 计算量。

**有 KV 缓存**：只为新生成的 token 计算 → 长输出显著提速。

**代价**：更高的内存占用；随序列长度线性扩展。

**KV 缓存压缩研究**：
| 方法 | 核心思路 |
|---|---|
| 动态逐出 | 识别对注意力贡献最大的"重击者" token；逐出其余 |
| 基于模式的聚类 | 识别注意力模式；聚类重要特征 |
| 按层重要性排序 | 不同层需要不同缓存大小；在全局预算下二分查找 |

**代表性工作**：vLLMPA (Kwon et al., 2023)、ChunkKV (Liu et al., 2025j)、StreamingLLM、EpiCache

## 权衡与设计指导（§3.1.3）

| 属性 | 内部（权重） | 内部（潜态/KV） | 外部 |
|---|---|---|---|
| 访问速度 | 快 | 非常快 | 较慢（检索） |
| 容量 | 固定 | 有限（线性） | 高度可扩展 |
| 更新成本 | 非常高（重新训练） | 短暂（重置即失） | 低（插入/删除） |
| 跨会话持久性 | 是 | 否 | 是 |
| 噪声风险 | 低 | 低 | 高（随规模增长） |
| 最适合 | 稳定的通用知识 | 会话内状态 | 长期经验存储 |

### 混合架构（当前最佳实践）

- **参数化记忆**：内在事实和世界知识
- **潜态记忆**：会话内快速短期推理
- **外部记忆**：跨会话的可扩展经验存储

这反映了从**静态持久知识**到**动态自适应**上下文管理的更广泛转变。

---

## 研究空白与扩展方向

- **噪声管理**：随着外部记忆增长，检索质量下降 —— 剪枝/摘要方法的活跃研究方向
- **混合协调**：参数化记忆与外部记忆在推理时如何交互？如何决定信息存于何处？
- **记忆效率**：KV 缓存压缩仍是活跃研究方向（ChunkKV、EpiCache）
- **持续学习 + 智能体**：将参数权重更新与外部记忆管理结合，同时避免灾难性遗忘
