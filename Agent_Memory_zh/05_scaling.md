# 规模扩展：记忆、上下文与环境

> 论文第 6 节（第 30–33 页）

随着 LLM 和智能体在日益真实的场景中部署，上下文沿**三个规模轴**快速增长：
1. **交互时域** —— 多会话、跨月的工作流
2. **环境复杂度** —— 异构数据、工具状态、结构化制品
3. **系统规模** —— 多个智能体 + 人类协作者

---

## 核心问题

传统评估使用**上下文受限的简单环境** —— 智能体在短时域内与合成用户交互。这导致：
- 高基准分数（MMLU 90%+、MATH、SWE-bench 部分）
- 但智能体在需要长期适配、用户个性化或任务特定技能复用的**现实世界场景中失效**

> "实用性差距：在现有基准上取得高分的智能体，在开放式、动态环境中往往难以展现长期适配、用户个性化或任务特定技能复用的能力。"

---

## 6.1 上下文受限的简单环境

大多数现有基准处于此类别 —— **受控、情节式、重置中心**：

| 基准类型 | 示例 | 记忆压力 |
|---|---|---|
| 问答（冻结知识） | SQuAD、HotpotQA、KILT、MS MARCO、TriviaQA | 极低 —— 静态检索，无跨查询状态 |
| 工具使用/网页 | ToolBench、WebShop、WorkArena | 低 —— 固定 API，抽象掉了认证/失败/长期状态 |
| 序列/交互式 | ScienceWorld、ALFWorld、TextWorld、Jericho | 中等 —— 情节式，但以重置为中心 |
| 代码/软件工程 | APPS、SWE-bench、SWE-BenchPro、SWE-Lancer | 中等 —— 任务内状态，但无跨情节知识 |

**这些基准的缺失**：跨情节的知识积累、长时域一致性、偏好演化。

**后果**：评估无法衡量记忆在鲁棒现实部署中的关键作用。

---

## 6.2 上下文爆炸的现实世界环境

现实世界的部署使智能体面临**多轴上下文爆炸**。

### 6.2.1 随交互规模扩展

#### 用户-智能体长时域对话

早期轮次的信息（用户偏好、身份、约束）可能不会立即产生影响，但后来会成为关键。

**有限上下文窗口下的失效模式**：
- 早期上下文遗忘
- 随交互长度增加的渐进式上下文漂移

**核心洞察**：交互历史不是静态输入，更像是一个**随时间演化的潜在内部状态**。

**现有缓解措施**（各有权衡）：
| 策略 | 方法 | 局限性 |
|---|---|---|
| 滑动上下文窗口 | 丢弃最旧的上下文 | 早期信息永久丢失 |
| 启发式截断 | 保留最近的 N 个 token | 丢弃可能至关重要的早期上下文 |
| 摘要式记忆 | 压缩 → 核心摘要 | 压缩可能丢失细粒度细节 |
| 用户档案/人格记忆 | 显式的长期结构 | 可能无法捕获完整的交互细节 |

#### 多轮工具使用上下文爆炸

除对话历史外，基于工具的智能体还必须保留：
- 工具输入和输出
- 中间执行状态（在后续步骤中被反复引用）
- 规划轨迹 + 反思性推理（ReAct 风格）

**上下文大小增长**：在 ReAct 风格的智能体中，随交互长度呈线性乃至**指数级**增长。

**挑战**：简单地移除/压缩工具轨迹会破坏推理与行动步骤之间的因果依赖。

---

### 6.2.2 随环境复杂度扩展

随着环境变得复杂，上下文必须容纳：
- 异构数据模态
- 异步工具交互
- 协议/权限受限的外部状态

将结构化制品（API 响应、文件、数据库、日志、配置）**简单平铺**为 token 序列会：
- 低效（token 数量过多）
- 丢失结构（模式/来源/时间有效性丢失）

**后果**：记忆从隐式的提示积累转变为**显式的系统级组件**，负责结构化的上下文管理。

#### 外化记忆接口

外化记忆支持：
- 模式感知检索
- 版本控制
- 有针对性的编辑
- 访问控制

关键协议：
- **模型上下文协议**（Anthropic, 2024）—— 定义查询语义、更新策略、范围边界
- **Claude Agent Skills**（Anthropic, 2025）—— 用于智能体-环境状态交互的技能导向抽象

#### 持久化需求

用户模型、系统配置和任务制品代表**持久状态**，必须：
- 跨会话保持一致
- 遵守治理约束（隐私、权限、回滚）

挑战：模式漂移、并发更新、路径依赖评估。

---

### 6.2.3 随环境数量扩展

#### 记忆作为跨多工具环境的接口

现实世界的智能体在多个异构环境间交替：
- 物理家庭环境（感知反馈、长时域行为）
- 数字网页环境（基于浏览器的信息检索）
- OS/App 环境（跨应用的 GUI 交互）

记忆必须：
- 保留、区分并协调环境特定的状态
- 维护环境状态的结构化、独立表征
- 支持长时域推理，同时不突破上下文窗口

#### 多人-智能体系统记忆

随着智能体系统扩展到多个智能体 + 人类参与者：
- 有限上下文窗口下，简单的上下文共享会迅速碎片化
- 需要：与智能体对齐或半共享的记忆抽象，编码关系历史和智能体间依赖

**该领域的研究方向**：
- 保留专门视角同时支持共享基底集成的角色特定记忆模板
- 减少智能体间交换噪声的层次化/角色感知对话方案
- 用于集体决策的辩论协议（共识/多数投票）
- 用于组织关系依赖的隐式图结构表征

---

## 对记忆系统设计的影响

| 规模轴 | 记忆设计要求 |
|---|---|
| 长交互时域 | 持久用户模型；层次化摘要；时序一致性 |
| 复杂环境 | 带模式感知检索的外化记忆；版本控制；访问控制 |
| 多环境 | 跨环境状态分离；记忆即接口模式 |
| 多智能体 + 人类 | 权限感知路由；角色特定视图；冲突消解 |

## 记忆演进

```
简单基准智能体              真实世界部署的智能体
（上下文受限）        →      （上下文爆炸）
        │                              │
简单的交互日志          成熟的上下文管理系统
        │                              │
短期提示                  长期个性化
        │                              │
单环境                    跨环境状态管理
        │                              │
单智能体                  多智能体协调 + 人类协作
```

记忆从简单的缓冲区转变为**架构骨干**，支撑鲁棒的、长时域的、以用户为中心的智能体行为。
