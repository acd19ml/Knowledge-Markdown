# 基于训练的自演化

> 论文第 III-B 节（第 7–9 页）| 论文表 I

---

## 核心概念

基于训练的自演化通过使用由智能体自身生成或从已有知识中合成的数据来**更新模型权重**，从而提升智能体能力 —— 无需额外的人工标注。

两种子范式：
1. **合成驱动的离线演化** —— 从现有模型生成数据，离线训练
2. **探索驱动的在线演化** —— 在训练过程中与环境交互，从实时反馈中学习

---

## A. 合成驱动的离线演化

> "一种高效的引导工具，通过整合内部先验来启动模型 —— 但从根本上受基础模型初始容量的限制。"

智能体通过各种策略合成自己的训练数据，然后离线训练。

### 表 I：五类离线训练方法

#### 1. 指令合成
从模型自身生成多样化的指令-响应对。

| 系统 | 核心思路 | 重要成就 |
|---|---|---|
| **Self-Instruct**（Wang et al., 2023） | LLM 通过提示生成自己的指令数据集 | 数据高效对齐的基础 |
| **LIMA**（Zhou et al., 2023） | 少即是多 —— 1000 个精心筛选的样本可与大型数据集媲美 | 对齐中质量 > 数量 |
| 指令调整综述（Zhang et al., 2026） | 综合分类体系 | — |

**核心洞察**："SFT 记忆，RL 泛化"（Chu et al., 2025）—— 指令调整产生鲁棒的记忆但有限的泛化能力。

#### 2. 过程蒸馏
从更强的模型（或通过引导从模型自身）提取推理过程。

| 系统 | 核心思路 |
|---|---|
| **STaR**（Zelikman et al., 2022） | 引导推理：生成原理 → 过滤正确的 → fine-tuning |
| **Self-Play Fine-Tuning (SPIN)**（Chen et al., 2024） | 弱到强：自我博弈通过迭代 SFT 将弱模型转化为强模型 |
| **ReST-MCTS***（Zhang et al., 2024） | 通过过程奖励引导的树搜索进行自我训练 |

#### 3. 合成增强
将训练数据扩展到人工标注的边界之外。

| 系统 | 核心思路 |
|---|---|
| **Beyond Human Data**（Singh et al., 2023） | 将自我训练扩展超越人类标注数据的限制 |
| **LADDER**（Simonds & Yoshiyama, 2025） | 用于自我改进的递归问题分解 |
| 各类规模扩展工作 | 通过多样化采样策略进行数据增强 |

#### 4. 自我博弈
智能体与自身的不同版本竞争或协作以生成训练信号。

| 系统 | 核心思路 |
|---|---|
| **SPPO**（Wu et al., 2024） | 用于对齐的自我博弈偏好优化 |
| **SPIRAL**（Liu et al., 2025） | 零和博弈上的自我博弈 → 通过 MARL 激励推理 |
| **SPC**（Chen et al., 2025） | 通过对抗博弈演化的自我博弈批评者 |
| **ABSOLUTE ZERO**（Zhao et al., 2025） | 零外部数据下的强化自我博弈推理 |

**核心洞察**：自我博弈无需外部数据即可创建多样化的、自动课程化的训练信号。

#### 5. 智能体流水线演化
智能体优化自己的提示、工作流和代码流水线。

| 系统 | 核心思路 |
|---|---|
| **DSPy**（Khattab et al., 2023） | 将声明式 LM 程序编译为自我改进的流水线 |
| **SIRIUS**（Zhao et al., 2025） | 通过自我改进引导推理的多智能体系统 |
| **Agent Evolver**（Zhai et al., 2025） | 面向高效自演化智能体系统 |

## B. 探索驱动的在线演化

> "通过将 LLM 转变为主动智能体，超越数据上限，该智能体通过迭代试错发现新策略。"

智能体在**与环境交互的同时**进行学习，通过探索生成自己的奖励信号。

### 关键机制：环境作为训练场

```
智能体 → 采取行动 → 环境返回反馈
  ↑                       │
  └── 通过 RL 更新策略 ───┘
      （GRPO、PPO、RLVR）
```

环境提供**客观反馈**，防止自我强化偏差，并支持超越初始训练分布的策略发现。

### RL方法

| 算法 | 类型 | 关键属性 |
|---|---|---|
| **PPO**（近端策略优化） | 同策略 RL | 稳定，在 RLHF 中广泛使用 |
| **GRPO**（组相对策略优化） | 基于组的 RL | DeepSeek-R1 的核心算法 |
| **DPO**（直接偏好优化） | 离线 RL | 无需奖励模型 |
| **RLVR**（可验证奖励的 RL） | 验证引导 | 数学/代码领域 |

### 关键系统

| 系统 | 领域 | 关键创新 |
|---|---|---|
| **WebRL**（Qi et al., 2024） | 网页导航 | 面向网页智能体的自演化在线课程 RL |
| **WebAgent-R1**（Wei et al., 2025） | 网页任务 | 端到端多轮 RL 训练 |
| **RAGEN**（Wang et al., 2025） | 智能体任务 | 通过多轮 RL 理解自演化 |
| **Search-R1**（Jin et al., 2025） | 搜索 | 训练 LLM 用 RL 进行推理和搜索 |
| **R-Zero**（Huang et al., 2025） | 推理 | 从零数据自演化推理 LLM |

### 可验证的奖励信号

关键使能因素：**可自动验证的奖励**，无需人类评判。

示例：
- **数学**：符号求解器、Lean 证明器（AlphaProof）
- **代码**：测试套件、编译器、单元测试
- **网页任务**：任务完成检测
- **搜索**：检索文档的相关性

> "代码执行器、数学引擎或开放网络等外部环境提供客观信息 —— 环境充当防止自我强化偏差的关键修剪机制。"

## 关键张力：离线 vs 在线

| 维度 | 合成驱动的离线 | 探索驱动的在线 |
|---|---|---|
| 数据来源 | 合成/现有模型 | 实时环境交互 |
| 效率 | 高（批量训练） | 较低（顺序交互） |
| 上限 | 基础模型初始容量 | 环境复杂度 |
| 风险 | 模型崩溃 | 奖励黑客 |
| 安全性 | 更可控 | 可能发现漏洞 |
| 最适合 | 领域引导启动 | 开放式能力增长 |

> "离线是高效的引导工具；在线通过发现超出训练分布的新策略，超越数据上限。"

## 代表性训练方法总结

| 类别 | 方法 | 核心技术 | 权重更新 |
|---|---|---|---|
| 指令合成 | Self-Instruct、LIMA | 自生成指令 | SFT |
| 过程蒸馏 | STaR、SPIN | 推理引导 | SFT |
| 自我博弈 | SPPO、SPIRAL | 对抗/协作 | SFT/RL |
| 探索驱动 | WebRL、RAGEN | 环境交互 | RL（PPO/GRPO） |
| 验证驱动 | AlphaProof、TTRL | 形式化验证器 | RL |
